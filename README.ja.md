# Bridge Protocol
### 公理から導出される現象世界の未確定な証明

[English](README.md) | **日本語** | [Español](README.es.md)

「Bridge」は `DCWorldBridge`（WorldDC.lean）と `bridgeOpen`（Gate.lean）——自己維持と世界の立ち上がりを繋ぐ Lean の型。現象ゲートは常に `bridgeOpen`。`pass` にはなれない。

---

## このリポジトリは何か

Bridge Protocolは自己維持する系の圏論的理論を構築し、それを機械検証することを目的とするプロトコル:


```
議論 → 圏論的仕様 → Lean 4 証明 → Julia 実装
```

理論の中核となる構造:

- **感覚運動随伴 `α ⊣ σ`** — 行為と感覚のあいだのガロア接続。系が何に触れられ、
  何に触れ返されるかは、一つの構造の両面である。
- **閉包 `Φ` とその最大不動点 `νΦ`** — 自己維持は**最大**不動点として定式化される。
  つまり「生き続けること」であって、何かの最大化では**ない**。
- **蝶番条件 `Act ≠ ∅`** — 取りうる行為が常に少なくとも一つ存在する。系が世界から
  閉ざされることはない。
- **自己維持証明 `DC`** — 系が自身の動態のもとで自己を維持することの、機械検証可能な証人。
- **立ち上がる世界 `Wld`** — その系に**とって**の世界。外から与えられるのではなく、
  動きと感触の循環から浮かび上がる。身体が変われば世界も変わる。

全体を貫くのが厳格な二層規律である。個々の系は**対象層**（構造要件 M1〜M4 に従う）に
属し、発展・選択に関わる公理はすべて別の**メタ層**にのみ注入され、個体側へ
書き戻されることはない。

## このプロジェクトが主張**しない**こと

この節は理論そのものと同じくらい重要である。

- **意識の主張はしない。** 構造の記述が完成し、完全に検証されたとしても、
  「中に灯りが点いているか」——主観的経験があるか——は外から証明できない。
  理論はその問いに答えず、記述の外に可能性として残す。この誠実さは機械的に
  強制されている: マーカー `phenomenal_claim = :not_certified` は認証アーティファクト
  連鎖の一部であり設計上いかなる証明によっても昇格しない。
- **最適化の物語ではない。** 維持は最大不動点であって最大化すべき報酬ではない。
  対象層は外部set pointと到達可能な終対象を禁じる（要件 M4）。
- **暗黙の同一視をしない。** 開放系のviability（`viable`）と ERIE-C の自己維持証明
  （`DC`）は区別され続ける。両者の同値は未証明であり、決して仮定されない。

## 検証方法論

このリポジトリの数学的主張はすべて**検証点**（VP）として単一の台帳
[specs/ledger.toml](specs/ledger.toml) に登録され、ゲートを通って状態遷移する:

```
proposed ──G1──▶ formalized ──G2──▶ bound ──G3──▶ implemented ──G4──▶ certified
```

- **G1** — Lean 4 形式化が typecheck する（`lake build`、`sorry` なし）。
- **G2** — Lean 宣言が contract test によって Julia シンボルに束縛される。
- **G3** — Julia 実装がテストを通過する。
- **G4** — contract が certificate catalog に登録され、依存グラフが検証される。

台帳に意味を与える規則が二つある。第一に主張が `certified` と記されるのは
[logs/gates/](logs/gates/) 配下に実際のゲートログが存在する場合**のみ**であり、
そのログは証拠としてコミットされる。第二に、**可視ギャップ原則**: 圏論的動機は
あるが Lean 証明がない主張は `certified` 未満として可視化され続け、黙って消される
ことも黙って信じられることもない。

2026年7月11日時点で台帳は 47 の検証点を追跡しており、うち 45 が certified、
2 が提案中である。

## リポジトリ構成

| パス | 内容 |
|---|---|
| [formal/ERIEC/](formal/ERIEC/) | Lean 4 形式化（47 モジュール: 随伴・閉包・蝶番・DC・系譜・豊かさ・増殖 ほか） |
| [src/](src/) | Julia 参照実装（`ERIEC.jl` パッケージ） |
| [test/](test/) | Julia テスト（Lean–Julia contract test を含む） |
| [specs/](specs/) | 検証点台帳と凍結 statement ファイル |
| [category/](category/) | 圏論的作業文書 |
| [docs/](docs/) | 理論概要・要件・設計文書 |
| [logs/gates/](logs/gates/) | ゲート証拠ログ（すべての `certified` を裏付けるビルド・テスト出力） |

`docs/` と `category/` の作業文書の多くは日本語で書かれている。Lean と Julia の
ソースが言語非依存の中核である。

## 検証の再現

下記ライセンスを許諾する: ソースを読み、コンパイルし、
記載された結果を独立に再現すること。

```bash
# Lean 証明（toolchain は ./lean-toolchain に固定）
lake build

# Julia 実装と Lean–Julia contract test
julia --project=. -e 'using Pkg; Pkg.instantiate(); Pkg.test()'
```
