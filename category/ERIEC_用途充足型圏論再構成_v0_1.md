## 1. 再構成の結論

ERIE-C の用途を一つの強い公理系へ押し込めるのではなく、次の共通中核と独立保証プロファイルへ分解する。

```text
                         ┌─ Audit      実装から模型への忠実性・来歴・非干渉
ERIE-C Core ─ OpenTrace ├─ Viability  有限内的地平・回復・開放ライブネス
                         ├─ Generative 系譜・新規性・非周期性
                         └─ Translation 理論間翻訳・賭けの局在化
```

強い制約を課す対象は「すべての系」ではなく、各保証ラベルを名乗るための証明義務である。プロファイルは直線的な成熟段階ではなく、原則として独立に合成される。

## 2. 共通中核: ラベル付き開放遷移と経路圏

### 2.1 開放遷移グラフ

状態型 `S` とラベル型

```text
Label := internal | coupling | meta
```

を持つグラフ

```text
G := (S, step ⊆ S × Label × S)
```

を**開放遷移グラフ**と呼ぶ。`[DEF]`

- `internal`: 指定された系境界の内部更新。
- `coupling`: 系境界を横切る物質、信号、資源、摂動との結合。
- `meta`: 個体群選択、設計、学習条件設定など、対象層の外側で起こる操作。

`coupling` と `meta` を分離する。外部と交換することは、外部の評価目標を個体の作動決定へ書き戻すことを意味しない。

### 2.2 経路圏

`Path(G)` を `G` が生成する自由圏とする。対象は `S`、射は有限ラベル付き経路、恒等射は長さ 0 の経路、合成は経路連結である。`[DEF]`

ラベルがすべて `internal` である射が生成する部分圏を `Path_i(G)`、`internal` と `coupling` が生成する部分圏を `Path_o(G)` とする。`meta` は対象内の開放実行に混入させず、後述のメタ圏に置く。

この定式化では「時間」は先に与えられた外部クロックではなく、生起した射の合成順序から得られる。実験上の秒、step 番号、timestamp は `Path(G)` から外部順序への順序保存 readout として別に置く。

### 2.3 許容部分圏

状態述語 `A : S → Prop` が定める充満部分圏を

```text
𝒜 ↪ Path_o(G)
```

と書く。同型な状態を同じ構造状態として扱う場合、`𝒜` は同型で閉じた replete subcategory とする。`[DEF]`

V5.1 への適用では `A(s)` の候補は `DC(𝔽,s)` だが、本節の定義は `DC` に依存しない。

## 3. 有限内的地平と開放自己維持

### 3.1 有限内的地平

```text
FiniteInternalHorizon(G,𝒜) :⇔
  ∀s∈𝒜, ∃N,
    ∀p : s ⟶ t in Path_i(G), length(p) ≥ N → t∉𝒜
```

を有限内的地平と呼ぶ。`[DEF]`

これは内部更新を十分長く続ければ許容部分圏を離脱するという普遍経路条件である。無限の純内的実行について結論する場合は、内部後続の全域性または実際の無限軌道を別に要求する。

V5.1 定理 5.4 は、有限 `C`、Sig-2、R2′、`h_int` のもとでこの条件の具体例を与える。`[OLD-THM]`

### 3.2 開放実行

開放実行を、関手

```text
x : ℕ_path ⟶ Path_o(G)
```

または同値な対象列・射列

```text
s₀ -f₀→ s₁ -f₁→ s₂ -f₂→ ⋯
```

として定める。各 `f_n` のラベルは `internal` または `coupling` である。`[DEF]`

### 3.3 二つのライブネス

存在的反復帰還性を

```text
Live∃(G,𝒜) :⇔
  ∃x, ∀n, ∃m≥n, x(m)∈𝒜
```

とする。これは凍結文書 `W₆` の抽象形であり、時相論理では `◇□` でなく `□◇𝒜` に対応する。`[DEF]`

頑健な反復帰還性を

```text
Live∀(G,𝒜,Fair) :⇔
  ∀x, Fair(x) → Maximal(x) → ∀n, ∃m≥n, x(m)∈𝒜
```

とする。`[DEF]`

`Live∃` は「維持できる実行がある」、`Live∀` は「公平な実行なら維持される」を表す。製品の自己維持保証には原則として後者または確率版が必要であり、前者だけを頑健性と読んではならない。

### 3.4 回復包絡と不可逆部

`𝒜` を含む指定部分圏 `ℬ` を**回復包絡**とし、`ℬ∖𝒜` を一時的離脱領域とする。`[DEF]`

```text
Recover∃(ℬ,𝒜) :⇔
  ∀s∈ℬ, ∃a∈𝒜, Hom_Path_o(G)(s,a) ≠ ∅

Recover≤k(ℬ,𝒜) :⇔
  ∀s∈ℬ, ∃a∈𝒜, ∃p:s⟶a, length(p)≤k
```

V5.1 の

```text
↓V := {s | ∃v∈V, s→*v}
K  := S∖↓V
```

では `↓V` が最大の存在的回復可能領域、`K` が不可逆部である。従って `K` からの回復を追加公理として要求してはならない。`K` からの回復を許したい場合は `K` の定義を変更するのでなく、別の回復包絡 `ℬ` と不可逆部を再指定する必要がある。

### 3.5 両立定理

**定理 3.1** `FiniteInternalHorizon(G,𝒜)` と、無限純内的実行の存在のもとで、その純内的実行は `𝒜` 反復帰還性を持たない。`[THM]`

> 証明: 初期状態に対する有限地平 `N` 以後はすべて `𝒜` 外なので、`∀n∃m≥n, x(m)∈𝒜` に `n=N` を入れると矛盾する。∎

**定理 3.2** `FiniteInternalHorizon(G,𝒜)` と `Live∃(G,𝒜)` は両立可能である。`[THM]`

> 証明: 凍結文書の外的補給付き循環模型 `M↻` が証人になる。純内的枝は有限地平を持ち、結合射を含む循環実行は `𝒜` を無限回訪れる。これは既存核から全模型の `Live∃` が導かれるという主張ではなく、二命題の無矛盾性である。∎

### 3.6 内生的回復の抽象的意味

内生的回復は「外部入力なしの回復」ではない。次の組として定義する。`[DEF]`

1. 回復経路は `coupling` 射を含んでよい。
2. 許容判定 `A` と回復方向の選択は対象内部の状態・閉包構造から計算される。
3. 外部の評価値、設計者の score、目標状態を変更しても、対象内部入力が同じなら回復選択トレースは変わらない。

この第 3 条件は §5 の非干渉として形式化する。外部は回復の可能条件を供給できるが、到達すべき set point を個体へ注入しない。

## 4. 終対象、周期、生成を分離する

### 4.1 終対象ガード

V5.1 の `NoT*` は、指定された求めの到達薄圏が終対象を持たないことのガードである。`[OLD-THM]`

注意すべきは、状態到達圏と生起列の圏を混同しないことである。同じ状態を反復する実行でも、生起を別対象として保持する自由経路圏は異なる構造を持ちうる。監査対象がどちらかを証明証書に明記する。

### 4.2 最終周期性

開放実行 `x` の対象列について、弱い最終周期性を

```text
EventuallyPeriodic_obj(x) :⇔
  ∃N, ∃p>0, ∀n≥N, x(n+p) ≅ x(n)
```

と定める。`[DEF]`

射を含む図式としての強い最終周期性は、上の同型族が各遷移四角形を可換にすることまで要求する。これを `EventuallyPeriodic_diag(x)` とする。`[DEF]`

```text
Aperiodic_obj(x)  :⇔ ¬EventuallyPeriodic_obj(x)
Aperiodic_diag(x) :⇔ ¬EventuallyPeriodic_diag(x)
```

対象ごとの非同型だけで射の反復を判定したことにしてはならない。

### 4.3 生成性

構造量を外部から読む関手

```text
r : 𝒢 ⟶ P
```

を置く。`P` は前順序圏、`𝒢` は §7 の生成・系譜圏である。`r` は対象内 action loop へ write-back しない read-only 観測である。

系譜 `g₀→g₁→⋯` の生産性候補を次で定める。`[DEF]`

```text
Productive_r(g) :⇔ ∀b∈P, ∃n, b < r(g_n)
```

`P=ℕ` なら、これは新規性尺度の非有界性である。別候補として、任意の `N` 以後に過去のどの対象とも非同型な対象が現れることを要求してもよい。どの `r` が科学的に妥当かは `[CNJ]` であり、M1–M4 から導出しない。

### 4.4 独立性

**定理 4.1** 次の含意はいずれも一般には成立しない。`[THM]`

```text
NoT*       → Live∃
Live∃      → NoT*
NoT*       → Aperiodic
Live∃      → Productive
Transport  → Productive
```

> 反例概略:
>
> - `NoT* → Live∃`: `𝒜` を一度も訪れない終対象なしの鎖。
> - `Live∃ → NoT*`: 一状態自己ループ。反復帰還するがその状態が終対象。
> - `NoT* → Aperiodic`: 相互到達しない二つの周期成分。全体に終対象はないが各実行は周期的。
> - `Live∃ → Productive`: 同一許容状態を反復する実行。
> - `Transport → Productive`: `Transport` は核の同型を要求するため、指定した同型不変量について新規性を作らない。∎

## 5. Audit プロファイル: 実装から形式模型への忠実性

### 5.1 二つの圏

`Impl` を実行可能系と観測可能遷移の圏、`Spec` を有限または有効表示された開放遷移模型の圏とする。`[DEF]`

期待する抽象化を

```text
Q : Impl ⟶ Spec
```

と書くが、その存在と正しさは既存核から導かれない。`[OBL]`

監査結果は常に組

```text
(implementation fingerprint, Q version, abstract model, assumptions, certificate)
```

へ結び付ける。模型単体の合格を実装全体の合格と表示してはならない。

### 5.2 may/must 抽象化

安全性検査とライブネス検査では必要な抽象化方向が異なる。

- `may` 抽象: 具体遷移を過大近似する。具体遷移は必ず抽象遷移へ写る。
- `must` 抽象または witness: 抽象的な存在経路に、対応する具体経路を持たせる。

状態写像 `q:S_I→S_M` に対し、may 健全性を

```text
s →_I t  ⇒  q(s) →*_M q(t)
```

とする。`[DEF]`

`NoT*` を実装へ逆輸送するには、監査対象状態領域上で `q` が全射であり、禁止ラベルを消去せず、到達を保存することが必要である。余分な抽象状態を加えるだけでは、終対象不在の証明が偽陽性になりうる。

存在的ライブネスは may 抽象上の経路だけでは保証できない。抽象経路が具体実行へ持ち上がる path-lifting certificate、または具体 lasso/無限実行を生成する有限証書を要求する。`[META]`

### 5.3 条件付き監査健全性

**定理 5.1** `q` が監査領域上で全射、到達保存、禁止ラベル保存を満たすとする。抽象模型の監査領域が `NoT*` を満たすなら、具体監査領域も `NoT*` を満たす。`[THM]`

> 証明: 具体側に全状態から到達可能な `t` があれば、全射と到達保存により `q(t)` は抽象側の全状態から到達可能となり、抽象側 `NoT*` に矛盾する。∎

この定理は完全なソースコード一般について「隠れた目標がない」と判定するものではない。`q` が覆う実装領域と観測境界についてだけ成立する。

### 5.4 証明証書圏

証書を、単なる真偽値でなく依存関係を持つ対象として扱う。証書対象は少なくとも次を持つ。`[DEF]`

```text
Claim        検査した閉論理式
Subject      実装 fingerprint / artifact hash
Abstraction  Q と模型の版
Dependencies 使用した公理・定理・数値仮定
Checker      検査器識別子
Trust        Lean core / shell / floating-point / acquisition の境界
Scope        状態領域、時間範囲、有限性仮定
Result       proved | refuted | bounded | unknown
```

証書射は、依存関係を保存する変換またはより強い証書への精密化とする。既存 certificate envelope はこの圏の実装候補だが、現時点では Julia 数値実装と Lean 意味論の完全同値を証明していない。

### 5.5 目標来歴と非干渉

外部入力を次の effect label に分ける。`[DEF]`

```text
resource   維持に利用可能な交換
signal     環境との観測的結合
goal       到達すべき状態・報酬・評価基準
selector   個体群または設計者側の評価状態
```

保護 sink を

```text
Sink := {νΦ, 𝐕, seeking diagram, action selection}
```

とする。`goal` と `selector` から `Sink` への情報流を禁止し、`resource` と `signal` は型付けされた coupling port を介して許可する。

非干渉を

```text
NonInterference :⇔
  ∀g₁ g₂ i,
    sameIndividualInput(i) → protectedTrace(g₁,i)=protectedTrace(g₂,i)
```

とする。`[DEF]`

これは「定義の引数に外部目標が書かれていない」より強い。alias、closure、global state、control dependence を含む意味論的情報流を対象にする。

静的 taint 検査と動的 metamorphic test は有用な近似だが、一般には完全証明ではない。完全保証を名乗る場合は effect system、情報流型、または二実行非干渉証明を要求する。`[META]`

**定理 5.2** M4 が保護トレースだけで決まる述語であり、初期 selector で保護トレースが M4 を満たすとする。selector に関する `NonInterference` が成立すれば、任意の selector でも M4 は成立する。`[THM]`

> 証明: `NonInterference` により任意の selector の保護トレースは初期 selector のものと等しい。M4 はそのトレースについて外延的なので保存される。∎

逆は成立しない。M4 が偶然保存されても selector の情報が `𝐕` や action へ漏れている可能性がある。

## 6. Viability プロファイル: 回復とライブネス

### 6.1 必須契約

Viability プロファイルは次を別々に証書化する。`[DEF]`

1. `FiniteInternalHorizon`: 純内的反復の有限地平。
2. `Recover`: 指定回復包絡からの帰還可能性。
3. `Live∃` または `Live∀`: 開放実行の反復帰還。
4. `NoGoalWriteBack`: 回復方向への外部 goal/selector 非干渉。
5. `Boundary`: 回復包絡、不可逆部、監査対象境界の明示。

`FiniteInternalHorizon` だけを viability と呼んではならない。それは precariousness の証書であって、持続の証書ではない。

### 6.2 公平性

`Live∀` は coupling 射が存在するだけでは導けない。例えば、回復射が永久に選択されない実行が残る。そこで用途に応じて次のいずれかを追加する。`[AX]`

- 弱公平性: 継続的に可能な回復射はいつか発火する。
- 強公平性: 無限回可能になる回復射は無限回発火する。
- 確率公平性: 回復可能状態で回復経路が正の下限確率を持つ。
- スケジューラ証明: 実装された scheduler が期限内に回復射を選ぶ。

公平性の種類は証書へ記録する。公平性を暗黙の物理法則として扱わない。

### 6.3 臨界指標の地位

V5.1 定理 6.8 により `χ(L)=|1-‖T‖²|` は計算可能である。`[OLD-THM]`  
しかし `χ` が崩壊距離または回復時間を表すことは予想 6.9 であり、層間整合にも依存する。従って証書では次を分ける。

```text
SpectralComputed  : χ を計算した
CollapseCorrelated: 指定データ上で崩壊指標との相関を検証した
PredictiveBound   : χ から崩壊時刻・回復時間の誤差境界を証明した
```

現状から無条件に名乗れるのは第一段だけである。

## 7. Generative プロファイル: 保存射と生成事象

### 7.1 `OpenDyn`

二時間尺度を抽象化した開放系を

```text
A := (Q_fast, Q_slow, Q_ext, step_A)
step_A : Cfg(A) → Set(Cfg(A))
Cfg(A) := Q_fast × Q_slow × Q_ext
```

とする。`[DEF]`

射 `f:A→B` は構成写像 `Cfg(f)` と遷移シミュレーション

```text
image(Cfg(f), step_A(c)) ⊆ step_B(Cfg(f)(c))
```

を満たすものとする。恒等射と合成で閉じるため圏 `OpenDyn` を得る。`[THM]`

等号を要求する厳密可換射は部分圏をなす。数値近似による lax 射は誤差合成則を別途持たせる。

### 7.2 能力を構造追加として表す

```text
OpenDyn
  └─ Adaptive       slow state を変更できる
       └─ Viable    指定部分対象を前向き保存する
            └─ Replicative  分岐・継承・変異を持つ
                 └─ Evolutionary 集団遷移を持つ
```

を忘却関手で結ばれた構造追加として置く。`[DEF]`

通常遷移、個体内適応、親子分岐、集団選択を同じ `step` と呼ばない。特に `¬A→A` という状態回復を、新しい個体または構造の生成と同一視しない。

### 7.3 保存射と生成事象

既存の `Transport` や `OpenDyn` の厳密可換射は、指定された意味論を保存する。新しいトポロジーが旧トポロジーと可換しない場合、それは失敗した保存射ではなく、別種の**生成事象**として扱う。

この二種類を一つの Hom に押し込めず、次の二重構造を設計候補とする。`[DEF]`

```text
objects:    開放系
vertical:   振る舞い保存・精密化射
horizontal: 変異・分岐・組立・系譜事象
2-cells:    viability、M4、安全性、来歴を保存したことの証書
```

厳密な double category/equipment の構成は `[OBL]` とする。第一段階では、保存射の圏 `OpenDyn` と生成関係 `branch` を分離した二ソート構造でよい。

### 7.4 豊穣化と個体内 telos の分離

read-only な構造量 `r` について

```text
ProducesRicher(A,r) :⇔
  ∀parent, viable(parent) →
    ∃child∈branch(parent), viable(child) ∧ r(parent)<r(child)
```

を定義する。`[DEF]`

これは変異空間内の到達可能性であって、実際にその子が選ばれることを含意しない。実選択には公平性または確率核についての追加仮定が必要である。`r` を個体内 action loop に write-back すると外部 set point になりうるため、Audit プロファイルの非干渉を同時に要求する。

### 7.5 有限監査と無限生成の限界

有限状態模型上の `W₆`、到達、閉包、指定された `NoT*` は有限手続きで検査できる。一方、任意プログラムの無限ライブネス、非周期性、非有界生産性を完全に有限時間判定することは要求しない。`[META]`

Generative プロファイルは次のいずれかで認証する。

- 有界地平 `N` までの新規性証書。
- 帰納的不変量またはランキング／反ランキング関数。
- 余帰納的 simulation 証明。
- 生成規則自体が非有界系列を構成する証明。
- 経験的観測のみの場合は `observed` と表示し、`proved` と区別する。

## 8. Translation プロファイル: 理論の翻訳と審判範囲

### 8.1 理論圏と解釈

各理論を、署名、文、模型、充足関係を持つ対象として扱う。理論間の射は解釈または institution morphism とする。`[DEF]`

意識理論 `𝒯_i` から ERIE-C の構造言語 `𝓔` への翻訳を

```text
Tr_i : 𝒯_i → 𝓔
```

と書く。実際には単なる用語対応でなく、文の翻訳と模型の還元が充足を保存することを要求する。`[OBL]`

### 8.2 翻訳証書

翻訳は各主張を次へ分類する。`[DEF]`

```text
structural   ERIE-C の構造語彙へ翻訳できる
functional   FM 等の操作的述語へ翻訳できる
bridge       構造から現象への追加仮定を要する
untranslated 対応語彙または模型写像がない
```

翻訳が保守的であるとは、翻訳先で新たに証明された原言語文が原理論でも証明可能であることをいう。完全性・保守性を示していない翻訳を「共通座標」と呼ぶ場合、その範囲を限定する。

### 8.3 審判できること、できないこと

Translation プロファイルが審判できるのは次である。

- 主張が定義、定理、追加公理、予想のどれに依存するか。
- 現象的述語が構造的前提から導出されたか、橋仮定で追加されたか。
- 二理論の主張が同一の構造文へ翻訳されるか。
- 反模型がどの含意を否定するか。

次は審判しない。

- どの意識理論が経験的に真か。
- `Conscious` という構造述語を満たす系に現象があるか。
- 翻訳不能な概念が無意味か。

V5.1 定理 7.3 と §9.2 の橋は、この境界を保つ既存核として利用できる。`[OLD-THM]`

## 9. 保証プロファイルと認証ラベル

単一の `ERIEC-certified` 真偽値を廃し、次の直積型ラベルを用いる。`[DEF]`

```text
Core        : none | model | machine-checked
Audit       : none | bounded | simulation-sound | implementation-linked
Viability   : none | recoverable | existential-live | fair-live
Generative  : none | observed | bounded | productive-proof
Translation : none | glossary | satisfaction-preserving | conservative
Phenomenal  : not-certified
```

例:

```text
Core=machine-checked
Audit=bounded
Viability=existential-live
Generative=observed
Translation=glossary
Phenomenal=not-certified
```

この表示は、ある軸の強さを別軸へ流用することを防ぐ。特に `Core=machine-checked` でも `Audit=none` なら現実実装について何も保証しない。

## 10. 用途別の成立条件

### 10.1 AI システムの監査

最低条件:

```text
Core + Audit(simulation-sound以上)
+ provenance labels
+ goal/selector noninterference
+ NoT* の対象図式と状態領域の明示
```

この条件で初めて「ERIE-C の定義に照らした自律性適合監査」と呼べる。自律性一般または隠れた意図一般の検出器とは呼ばない。

### 10.2 人工生命・エージェント設計

最低条件:

```text
Core + Viability(fair-liveを推奨)
+ recovery envelope
+ scheduler/fairness certificate
+ implementation-linked Audit
```

生成・進化まで主張する場合だけ Generative を追加する。三状態崩壊模型は整合性証人であり、この用途の完成模型ではない。

### 10.3 意識科学の翻訳

最低条件:

```text
Core + Translation(satisfaction-preserving以上)
+ bridge assumptions の依存追跡
+ Phenomenal=not-certified
```

FM1–FM4 の名称対応だけなら `glossary` に留まる。

### 10.4 美学・実践

この用途は形式認証でなく反省的 readout である。`𝐕`、M4、有限地平を比喩的に利用できるが、個人の価値、健康、経験を測定したとは表示しない。追加の強公理は不要で、適用が非規範的解釈であることの明示だけを要求する。

## 14. 決定可能性と表示規律

### 14.1 有限模型で完全検査できる範囲

- 指定有限関係上の到達可能性。
- `DC` の有限展開可能部分。
- `NoT*`。
- `W₆` の有限 lasso 標準形。
- 有限証書に含まれる同型、非干渉表、依存関係。

計算量の主張は入力表現とアルゴリズムを固定して行う。「すべて有限時間でチェック可能」という一括表現は使わない。

### 14.2 一般には完全検査を要求しない範囲

- 任意プログラムの全実行ライブネス。
- 無限実行の非周期性。
- 非有界構造生成。
- 実装と数学模型の完全意味論同値。
- 構造から現象的経験への橋。

これらは証明証書付き、有限有界、経験的観測、未決のいずれかとして表示する。

## 15. 本再構成の中心命題

本体系が最終的に保証すべき形は、次の単一命題ではない。

```text
「この系は自律的であり、生きており、生成的であり、意識的である」
```

代わりに、独立な証書の束として次を返す。

```text
1. 指定された純内的動態は有限地平を持つ。
2. 指定された公平な開放実行は許容部分圏へ反復帰還する。
3. 回復方向は外部 goal/selector に非干渉である。
4. 求めの指定図式は終対象を持たない。
5. 指定された系譜は最終周期的でない、または新規性証書を持つ。
6. これらの模型は指定実装の遷移を、明記した範囲で忠実に表す。
7. 現象的経験については何も認証しない。
```

この分解によって初めて、崩壊定理は自己維持と矛盾せず、外部結合は外部目的の注入と混同されず、`NoT*` はライブネスや生成性の代用品でなくなり、形式模型の合格を実装の真実として過大表示しなくなる。
