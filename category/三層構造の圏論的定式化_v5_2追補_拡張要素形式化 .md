**ステップ 1(自然言語による直観の言語化)** ドラフト §X1–X6 がこの段階の成果物である。抽出された原始的直観は五つ: (a) 非退化性は二つの論理的構成(∃/∀)の不一致として現れる、(b) 自己維持の不安定性は支持失効による減衰の単調性のみを要する、(c) 離散層に閉じた身体構成は連続層への橋渡し負債を継承しない、(d) 保証は成分ごとの判定値として与えられ依存順序に沿って伝播する、(e) 構造的記述は自己同型で不変であり、対称な中心点を識別できない。

## §21. 隙間保存 — ∃/∀ 非対称からの非退化性(ドラフト §X1)

**ステップ 1(直観)** 存在的直像 `σ★` と普遍持ち上げ `σ∀★` は異なる論理的構成であり、非退化性は両者の不一致(隙間)の保存として現れる。両者の一致は剛性定理 1.3 の退化と同値である。

### 21.1 シグネチャ: σ 側の普遍持ち上げと第二の随伴

**定義 21.1(σ 側普遍持ち上げ)** 双対整合系 `𝕂`(定義 1.4)に対し

```text
α∀(N) := {e ∈ E | σ(e) ⊆ N}
```

を定める。`[DEF]`

**定理 21.1′(第二の Galois 接続)** `σ★(X) ⊆ N ⇔ X ⊆ α∀(N)`。順序圏の関手として `σ★ ⊣ α∀`。`[THM]`
> 証明: 両辺とも `∀e∈X, σ(e)⊆N`(定理 1.2 の双対)。∎

**定理 21.1″(定理 1.4 の双対・記号の非重複)** `hConv` のもとで `α∀(N) = {e | ∀a, e∈α(a) → a∈N}`(`α` から作る E 側の普遍持ち上げ)。すなわち `α∀` は定理 1.4 の `σ∀` と正確に双対の位置にあり、命名は v5.1 の規約(無星 = 逆側関係から構成した普遍持ち上げ)に従う。v5.1 の既存記号(`σ★, σ∀★, σ∀, Θ_w` のいずれ)とも定義・型が異なり、重複しない。`[THM]`
> 証明: `hConv` より `{a | e ∈ α(a)} = σ(e)`(定理 1.4 の証明と同一の書き換え)。型の相違は直接比較(`α∀ : Set(A) → Set(E)`、`σ∀★, σ∀ : Set(E) → Set(A)`、`Θ_w : Set(E) → Set(E)` かつ ∃ 合成)。∎

これにより随伴の二本鎖 `α★ ⊣ σ∀★` と `σ★ ⊣ α∀` が並立する。∃側構成 `σ★` は左随伴(和保存)、∀側構成 `σ∀★` は右随伴(積保存)であり、両者は**論理的役割からして別の側**に立つ。

### 21.2 退化の特徴づけ(隙間消失 = 剛性)

**定理 21.2(退化判定・hConv 形)** 双対整合系 `𝕂` について次は同値。`[THM]`

1. `σ★ = σ∀★`(`Set(E) → Set(A)` の写像として)。
2. `∀a ∈ A, |α(a)| = 1`。
3. `σ★` が `α★` の右随伴でもある(随伴三つ組 `α★ ⊣ σ★ ⊣ α∀` が成立する)。

> 証明: (1)⇒(2): `hConv` により `σ★(X) = {a | α(a) ∩ X ≠ ∅}`。`X = ∅` を代入すると `σ∀★(∅) = {a | α(a) = ∅}` と `σ★(∅) = ∅` の一致から `∀a, α(a) ≠ ∅`。次に `e ∈ α(a)` を取り `X = {e}` を代入すると `a ∈ σ★({e})`、一致より `α(a) ⊆ {e}`、よって `|α(a)| = 1`。(2)⇒(1): 一点集合は「交わる ⇔ 含まれる」。(1)⇔(3): 随伴の一意性(薄い圏では等式)により、`σ★` が右随伴 ⇔ `σ★ = σ∀★`。定理 1.3 との整合: `hConv` のもとで `σ` は `α` の逆関係に確定するため、(2) は剛性定理 1.3 の退化条件(`α` の一点値性)そのものである。∎

**定理 21.3(隙間の方向分解)** 各 `a ∈ A` について

```text
GapUp(a) :⇔ ∃X, a ∈ σ★(X) ∖ σ∀★(X)
GapDn(a) :⇔ ∃X, a ∈ σ∀★(X) ∖ σ★(X)
```

と置くと(`[DEF]`)、`hConv` のもとで

1. `GapUp(a) ⇔ |α(a)| ≥ 2`(多価性による上向き隙間)。
2. `GapDn(a) ⇔ α(a) = ∅`(空値による下向き隙間)。
3. したがって各 `a` は「隙間なし(一点値)」「上向き隙間(真の多価)」「下向き隙間(空値)」のちょうど一つに属し、`σ★ ≠ σ∀★ ⇔ ∃a, GapUp(a) ∨ GapDn(a)`。

`[THM]`
> 証明: (1) ⇐: `e ∈ α(a)` に対し `X := {e}` で `a ∈ σ★(X)`、`|α(a)| ≥ 2` より `α(a) ⊄ {e}` で `a ∉ σ∀★(X)`。⇒: `a ∈ σ★(X)` より `α(a) ∩ X ≠ ∅`、`a ∉ σ∀★(X)` より `α(a) ∖ X ≠ ∅`、二元は相異なる。(2) ⇐: 任意の `X` で `∅ ⊆ X` かつ `∅ ∩ X = ∅`。⇒: `α(a) ⊆ X` かつ `α(a) ∩ X = ∅` から `α(a) = ∅`。(3) 場合分けと定理 21.2。∎

**帰結** v5.1 義務 1.6(真に多価な `α` の充足例)の証人は、まさに `GapUp` の居住証人である。**非退化性は上向き隙間の非空として、公理でなく既存構成から定義・導出される**。ドラフト申し送り 2(「新公理を要さない可能性が高い」)は肯定的に確定した。

### 21.3 設計原則の定理化

**定理 21.4(同一視公理の退化強制)** 理論 `AX_stable` に等式公理 `σ★ = σ∀★` を追加した拡大理論の任意の模型は `∀a, |α(a)| = 1` を満たす。したがって同拡大理論のもとで義務 1.6 型の非退化証人は構成不能である。`[THM]`
> 証明: 定理 21.2 の (1)⇒(2) を各模型に適用。∎

**設計原則 21.5(隙間保存)** いかなる拡張構成においても、∃的構成(`σ★, α★` の側)と∀的構成(`σ∀★, α∀` の側)は**別個の記号として導入し、両者の一致を公理・フィールドのいずれとしても要求しない**。要求した瞬間に定理 21.4 により退化する。非退化性を「収束の禁止」という否定的公理で要請してはならない(欲しい結論の直接要請の禁止)。非退化性は定理 21.3 により隙間の居住証人として**構成的に**示す。`[META]`
> 注意: 本原則は §3 の閉包 `Φ_w = π★∘ρ★`, `Θ_w = α★∘σ★` が∃的合成であることを変更しない。∀側の合成作用素への置換を要求するものでもない(§15′ の禁止読み替え参照)。原則の内容は「二構成を**併置**し等置しない」ことのみである。

### 21.4 π/ρ 側の双対

**定理 21.6(双対系)** `hConvP` のもとで、`ρ★, ρ∀★(Y) := {a | π(a) ⊆ Y}`, `π∀(N) := {c | ρ(c) ⊆ N}` について定理 21.1′–21.4 の全主張が mutatis mutandis に成り立つ。`[THM]`
> 証明: `(E, α, σ, hConv)` を `(C, π, ρ, hConvP)` に置換する同型な議論。∎

## §22. 減衰構造 — 自己維持核の基質非依存化(ドラフト §X2)

**ステップ 1(直観)** 不安定性(M1 の precariousness)の核は「支持が失われれば構成素が実際に減衰する」という単調性のみであり、特定の基質を要求しない。既存の更新作用素 `Ψ_w` はその一実現である。

### 22.1 シグネチャと公理(フィールド)

**定義 22.1(減衰構造)** 定台前層系(定義 2.8)上の**減衰構造**とは、作用素の族 `dec_w : Set(C) → Set(C)`(`w ∈ W`)であって次のフィールドを備えるものをいう。

```text
d1(収縮)      : ∀w Y, dec_w(Y) ⊆ Y
d2(単調)      : ∀w, Y ⊆ Y' → dec_w(Y) ⊆ dec_w(Y')
d3(支持固定)  : ∀w Y, Y ⊆ Φ_w(Y) → dec_w(Y) = Y
d4(非支持厳減): ∀w Y, Y ≠ ∅ ∧ ¬(Y ⊆ Φ_w(Y)) → dec_w(Y) ⊊ Y
```

構造は `[DEF]`、d1–d4 は `[FLD]`。d1 より `dec_w(∅) = ∅` は射影である(規律 0.2-1 により定理を名乗らない)。

**定理 22.2(標準居住: Ψ が減衰構造)** `Ψ_w(Y) := Y ∩ Φ_w(Y)`(§3.1)は d1–d4 を満たす。`[THM]`
> 証明: d1 は交差。d2 は定理 3.1(`Φ_w` 単調)と交差の単調性。d3 は `Y ⊆ Φ_w(Y) ⇒ Y ∩ Φ_w(Y) = Y`。d4 は `y ∈ Y ∖ Φ_w(Y)` の存在から `y ∉ Ψ_w(Y)`。∎
> 注意(非重複): 合成 `Ψ_w ∘ Ψ_w` も d1–d4 を満たし(d3: `Ψ_w(Y) = Y` なら二回適用も `Y`。d4: `Ψ_w(Y) ⊊ Y` に d1 を追加適用)、一般に `Ψ_w` と異なる(有限反例: `C = {c₁, c₂}, ρ(c₁) = {a}, ρ(c₂) = ∅, π(a) = {c₂}` で `Ψ({c₁,c₂}) = {c₂} ⊋ ∅ = Ψ²({c₁,c₂})`)。したがって減衰構造は `Ψ_w` の別名・重複定義ではなく、`Ψ_w` を標準居住とする真の一般化である。

これにより減衰構造は空虚な構造体ではない(居住証明則 0.2-4 の即時充足)。

**定理 22.3(自己維持核の減衰不変性)** d3 を備える任意の減衰構造について `dec_w(νΦ_w) = νΦ_w`。より一般に、任意の後不動点 `Y ⊆ Φ_w(Y)` は `dec_w` の不動点である。`[THM]`
> 証明: `νΦ_w ⊆ Φ_w(νΦ_w)`(定理 3.2)に d3 を適用。∎(条件明示則: 本定理は d3 のみに依存する。)

### 22.2 崩壊定理の抽象化(M1 の基質非依存性の定理形)

**定理 22.4(抽象崩壊定理)‡** `C` 有限、Sig-2(公理 3.7)、R2′(公理 5.3)‡ のもとで、d1・d4 を備える任意の減衰構造 `dec` と任意の収縮的な E 側族 `decᴱ_w`(`decᴱ_w(X) ⊆ X`)について、一般化更新

```text
upd_dec(K, X, w) := (dec_w(K), decᴱ_w(X), δ(w, K))
```

は次を満たす: 任意の初期構成 `c₀ ∈ Conf` について、ある `n ≤ |W| + |C|` が存在し、すべての `n' ≥ n` で `upd_dec^{n'}(c₀)` の κ 成分は `∅`(恒久性込み)。`[THM]‡`(‡ は R2′ 継承のみ)
> 証明: 定理 5.4(1) の証明の `Ψ_w` への依存箇所を d1・d4 で置換する。第一相: κ 成分が非空かつ `w < ⊤` の間、R2′ により w 成分は厳密増加し、有限線形順序 `W` 上で高々 `|W| − 1` 歩で `⊤` に達する(途中で κ 成分が `∅` なら恒久性の議論へ)。第二相: `w = ⊤ > w*` では定理 3.8 より非空後不動点は存在しないから、任意の非空 `K` について `¬(K ⊆ Φ_⊤(K))`(さもなくば余帰納原理 3.2(i) で `K ⊆ νΦ_⊤ = ∅`)。d4 により各歩で厳密減少し、`C` 有限なので高々 `|C|` 歩で `∅`。`w = ⊤` は R2′ の非減少性により保存される。恒久性: d1 より `dec_w(∅) = ∅`。∎(条件明示則: 本定理は d1・d4・Sig-2・R2′・`C` 有限に依存し、d2・d3 には依存しない。)

**系 22.4′** `dec := Ψ`, `decᴱ := Ψᴱ` の代入(定理 22.2)により定理 5.4(1) を回収する。`[THM]‡`

**メタ命題 22.4″(基質非依存性の形式的内容)** 「M1 の充足に基質の生物性は不要」というドラフト X2.2 の主張の形式的内容は、次の事実に尽きる: 定理 22.4 は署名 `{Set(C), Φ_w, dec_w, δ, W}` のみで述べられ・証明され、基質を指示する記号は署名に現れない。基質語(付録 X′)への言及を要する主張はこれ以上のものであり、対象理論の文として書けない。`[META]`

### 22.3 収束単位の非必然性 = 実現の多重性

ドラフト X2.3(結合単位 `u` は自然な収束先だが必然ではない)の「自然に収束しやすい」の部分は収束進化に関する経験的主張であり、基質語彙なしには文にならないため対象理論から除外する(`[META]`)。理論側で形式化できるのは**非必然性**の側であり、それは述語プロファイルが実装を同型を除いて確定しない、という模型多重性文である:

**義務 22.5(実現多重性の証人)** 同一の判定プロファイル(少なくとも `hConv`・Sig-2・`DC` 証人の存在・FM1 証人の存在)を充足し、かつ `𝔎at` において互いに非同型な二つの核 `𝕂 ≇ 𝕂'` を構成せよ。証明方針: 義務 13.1a の極小模型(`|A| = 1`)と、義務 25.6 の対称二重化模型(`|A| = 2`)。同型射の成分は全単射(定理 2.5)なので濃度差により非同型。`[OBL]`

**メタ規律 22.6(非昇格の再掲)** 義務 22.5 の充足は「プロファイルは実装を定めない」ことの証明であり、「いかなる実装でもよい」「収束先が存在しない」ことの証明ではない。逆に、収束の必然性を主張する `[THM]` はどの節にも存在しない(ドラフト X2.3 の注意の継承)。`[META]`

## §23. 離散再帰身体 — 橋渡し負債の遮断(ドラフト §X3)

**ステップ 1(直観)** 身体構成を最初から離散層(`Set(C)` 上の順序論的構成と §17 の開放グラフ)に閉じて与えれば、§8 の離散↔連続橋渡し(界面宣言 6.3・設計命題 8.5)の未解決負債を継承しない。非退化な離散再帰の存在は証人構成の義務であり、形式基盤内で身体を定義すれば実装忠実度の翻訳層は自明化する。

### 23.1 離散一貫拡張と依存遮断

**定義 23.1(離散一貫拡張)** v5.1 上の拡張理論(定義・定理・義務の追加集合)が**離散一貫**であるとは、その署名が連続層の記号

```text
T, T†, L, fderiv, adjoint, Fix, E_η, Wld_η, χ, λ_max, Rep, IRep, 𝔉, FM4, Ā, Ē
```

のいずれも含まないことをいう(排除基準のみで定義し、含んでよい節の正のリストは与えない — §13 の参照模型義務のように離散・連続の両成分を持つ節があり、節単位の列挙は不正確になるため)。`[DEF]`
> 居住注記(規律 0.2-4): 本追補の §21・§22・§24 はそれ自体が離散一貫拡張の居住例である(署名は §F ステップ 3 参照。上記排除リストの記号を含まない)。

**メタ規律 23.2(橋渡し負債の遮断)** 離散一貫拡張の定理は、界面宣言 6.3(未式化、台帳 I-1)にも設計命題 8.5(`[CNJ]`)にも論理的に依存しない。根拠の階層: 署名上の非出現(定義 23.1)は**必要条件にすぎず、それだけでは依存遮断を保証しない** — 結論の文に連続層記号が現れなくても、その証明が連続層の定理を経由することは一般に可能である(極端には、他層が矛盾していれば爆発原理で離散命題も導ける)。遮断の実際の根拠は依存グラフの非連結性である: 当該拡張の Lean モジュールが `ERIEC.Sensitivity / ERIEC.World / ERIEC.WorldDC` を import せず、かつビルド生成物(`.olean`)の依存グラフに連続層モジュールへの経路が存在しないことを CI で検査する(§16 のモジュール分離に一致)。構文的依存関係の主張であるため、v5.1 定理 7.3 Lean 化注記と同じ方針でメタ規律として文書側に置き、理論内定理としない。`[META]`
> 注意: 本規律は §8 の負債を**解消**しない。負債を**継承しない設計領域**を確定するのみである。FM4・`Conscious`(FM4 を含む)は連続層記号を含むため離散一貫拡張の内部では使えず、離散一貫な分類は FM1–FM3 まで(FM3 は ‡)である。この制限の明示が本規律の対価である。

### 23.2 非退化離散再帰の証人義務

**メタ命題 23.3(単調退化の回避は無償でない)** 離散再帰構造はそれ自体では非退化性を保証しない。有限状態上の決定的反復は最終的に周期的であり(鳩の巣)、また定理 17.15 により終対象不在・帰還・非周期は相互に導出されない。したがって非退化な離散再帰の存在は個別の証人構成義務である(ドラフト X3.2 の `[OBL 候補]` を確定)。`[META]`

**義務 23.4(非退化離散再帰身体の証人)** 次を同時に満たす有限構成を与えよ: (i) 台核 `𝕂` は真に多価な `α` を持つ(義務 1.6 = `GapUp` 居住、定理 21.3)。(ii) その上の動的認証枠 `𝔻` から誘導される開放グラフ `G_𝔻`(定義 17.16)が `FIH(G_𝔻, A)` を満たす(定理 17.17 の適用対象)。(iii) 結合辺を含めて `PossibleLive` を満たす(帰還証人)。証明方針: `M_recur`(定理 17.12 の証人 1)の二状態骨格の各状態に、義務 13.2 の多価核の構成を載せ、結合辺 `b →ₑ a` を E5 と両立する同型核対の上に置く。`[OBL]`
> 注意(条件明示): (iii) は `Trans_e ≠ ∅` を本質的に要する。純内的経路のみでは FIH(ii) と帰還(iii) は両立しない(定理 17.11)。「離散再帰身体」は開放系であり、閉鎖系の看板で提示してはならない。

### 23.3 形式基盤上の実装同一性

**定義 23.5′(構成的シミュレーション射)** 定義 18.1 の `maps : ∀s k t, G.step s k t → Nonempty(Path_H(f₀(s), f₀(t)))` は経路を命題的に消去するため、`labels`(「maps が作る経路は k と同じ辺種別だけを使う」)を当該経路について型付けできず、定理 18.2 の合成が連結すべき具体的経路も構成的には取り出せない。そこで証人携行版を定める:

```text
OpenSimᶜ の射 f : G → H :=
  (f₀      : G.State → H.State,
   mapEdge : ∀ {s k t}, G.step s k t →
             { p : Path_H(f₀(s), f₀(t)) // PathUsesOnlyKind(p, k) })
```

`[DEF]` 恒等(一辺経路)と合成(像経路の連結。種別保存は連結の帰納法)で `OpenSimᶜ` は圏をなし、`Nonempty` 化と `labels` の命題化からなる消去関手 `OpenSimᶜ → OpenSim` により定義 18.1 の射が回収される。逆向き(`Nonempty` からの証人取り出し)は選択原理を要し `Equiv.ofBijective` 型の隔離に置く(定理 11.3 と同型の処置)。
> **v5.1 修理推奨(台帳 R-4 提案)**: 定義 18.1 の `maps/labels` を本 `mapEdge` 形へ差し替えることを推奨する。追加のみの原則により本追補は v5.1 を変更せず、凍結文書 §6-1 の R-3(E5 差し替え推奨)と同型の手続きとして台帳へ記録するにとどめる。差し替え後、本定義は定義 18.1 に吸収され消去関手は不要となる。定理 18.3(NoT* の監査健全性)は経路の存在のみを使うため、いずれの版でも成立し影響を受けない。

**定理 23.5(恒等監査写像)** 任意の開放グラフ `G`(Lean 内部の項として与えられたもの)について、恒等状態写像 `id : G.State → G.State` と一辺経路の割当て `mapEdge(e) := ⟨cons(e, nil), 単一辺の種別一致⟩` は `OpenSimᶜ` の射 `id_G : G → G` を与え(消去関手経由で定理 18.2 の恒等射を回収)、組

```text
AuditMap(G, G) := (q := id_G, initPres := 恒等含意, scopeSurj := 対角証人,
                   fingerprint := G の項それ自体, assumptions := ∅)
```

は §18.2 の `AuditMap` 定義の全フィールドを充足する。したがって形式基盤内で定義された身体については、**証書の `trustBoundary` を Lean core とする範囲において**、保証プロファイルの `Audit` 成分は追加の翻訳層なしに `implementationLinked` に到達する(範囲外の地位はメタ規律 23.6)。`[THM]`
> 証明: `initPres` は恒等写像で自明。`scopeSurj` は各 `m` に `i := m` を取る。`PathUsesOnlyKind(cons(e, nil), k)` は当該一辺の種別がもとの `k` そのものであることによる(定義 18.1 の `Nonempty` 版では、この最終段が当該経路について型付けできないため、本定理は定義 23.5′ の上で述べる — 条件明示則 0.2-5)。∎

**メタ規律 23.6(実装同一性の信頼境界)** 定理 23.5 の同一性は Lean core の信頼境界の**内側**でのみ成立する。証書(定義 18.8)の `trustBoundary` には Lean kernel・コンパイラ・実行基盤が引き続き残り、形式項と物理的実行体の同一性は本定理の範囲外である。「測定・翻訳層の不要化」(ドラフト X3.3)は正確には「`AuditMap` の `q` が恒等に取れる」ことであり、信頼境界の消滅ではない。`Core = machineChecked` かつ `Audit = implementationLinked` でも、境界外の主張には従前どおり別途の証書を要する(§18.2 の `[META]` の継承)。`[META]`

## §24. 判定射と保証プロファイルの伝播構造(ドラフト §X4)

**ステップ 1(直観)** 各保証成分への判定は五値 `{pass, fail, unk, open, na}` の表示を持つが、その実体は証拠携行である: `pass` は充足の証明を、`fail` は不充足の証明(反証)を携え、どちらの証明もなければ `unk` にとどまる。前提未充足(`fail`/`na`)のみが依存順序に沿って `na` を伝播させる。現象性成分は `pass` も `fail` も返し得ない。

### 24.1 シグネチャ

**定義 24.1(判定値型)**

```text
Gv := {pass, fail, unk, open, na}
```

`[DEF]` 値の意味(非規範注釈): `pass` = 充足の**証明がある**。`fail` = 不充足の**証明(反証)がある**。`unk` = いずれの証明もない(証明の不在は反証ではない)。`open` = 前提層には到達したが、**本仕様内では**橋が認証されていない(橋 A→B 型 `[CNJ]`。将来の追加橋・拡張理論での公証可能性を排除しない — 定理 7.3 が示すのは `AX_stable` からの無解釈述語への一様導出不能性であって、あらゆる理論での不可能性ではない)。`na` = 依存前提の不充足(`fail`/`na`)による対象外。
> 記号注記(衝突回避): 型名は `Gv` とし、§17・§23 の開放グラフ変数 `G` と区別する。また `Gv` は定義 18.8 の証書帰結型 `result(proved | refuted | bounded | observed | unknown)` とも、§20.2 の成分列挙型とも**別の型**である。三者の間の対応(例: 各成分の証書群から `raw_i` を算出する規則)は証書側のデータであり、本追補は同一視も自動変換も導入しない(義務 20.2 の「別軸自動昇格の禁止」と整合)。

**定義 24.1′(証拠携行判定型)** `Gv` は表示用の五値列挙にすぎず、判定の実体は証拠携行型とする。成分 `i` の形式的内容を `P_i : Prop`(当該成分の証書 claim、定義 18.8)とし、

```text
RawEv(P) :=                            (生判定・証拠携行・na 構成子なし)
  pass(prf : P)                        充足の証明
| fail(ref : ¬P)                       不充足の証明(反証)
| unk(r : UnknownReason)               いずれの証明もない(理由の記録)
| open(o : OpenObligation)             本仕様内で橋が未公証(橋義務の記録)

GateEv(P) := RawEv(P) ⊎ {na(d : DepFail)}     (判定結果・依存失効証人つき)
eraseEvidence : GateEv(P) → Gv                (証拠を捨て表示値へ落とす)
```

`[DEF]` 本型により次が**型水準で**強制される: (i) 根拠なしに `pass`/`fail` を書き込む口(自己申告欄)は存在しない — `pass` は `P` の証明項を、`fail` は `¬P` の証明項を要求する。(ii) 「Lean で証明が書けなかった」は `unk` であり `fail` ではないから、未証明の成分が後続成分を `na` に落とすことはない(偽陰性の遮断)。§26 の係留規律は本型により Lean の型検査へ内在化される。
> Lean 注記: `open` は Lean 4 の予約語のため、実装では構成子を `bridgeOpen` 等に改名する(表示名のみ `open` を維持)。

**定義 24.2(依存図式と生判定)** 保証成分の有限添字集合 `Idx`(§20.2 の六成分 `{Core, Audit, Viability, Generative, Translation, Phenomenal}` を標準とする)上に、依存関係 `≺ ⊆ Idx × Idx` と各成分の**生判定** `raw_i` を与える:

```text
𝒢 := (Idx, ≺, P, raw)
≺    : 有限・非循環(推移閉包 ≺⁺ が非反射)      [FLD]
P    : ∀i ∈ Idx, P_i : Prop                     (成分 i の形式的内容 = 証書 claim)
raw  : ∀i ∈ Idx, raw_i : RawEv(P_i)             (na 構成子なし — 型で保証)
制約 : RawEv(P_Phenomenal) は open 構成子のみを持つ
```

構造は `[DEF]`、非循環性は `[FLD]`。`raw_Phenomenal` の型制約は §20.2 の `Phenomenal : notCertified` の判定値表現であり、`pass`・`fail` のいずれも構成子として存在しない(型水準の禁止。反証側も認証しないのは、橋 A→B が `[CNJ]` であって肯定・否定のどちらも本仕様の証明対象でないため)。標準依存図式は `Core ≺ Audit`, `Core ≺ Viability`, `Core ≺ Generative`, `Core ≺ Translation`, `{Audit, Translation} ≺ Phenomenal` とするが、`≺` は用途ごとのデータであり本標準は例示である。
> 記号注記(衝突回避): 添字集合は `Idx` とし、§11.2 の索引型 `I`・§18.2 の実装遷移系 `I` と区別する。判定枠は `𝒢`(script)とし、設計命題 8.5 の関手 `𝔊`(Fraktur)とは別記号・別対象である(`𝒢` は圏でも関手でもなく有限データの組)。

**定義 24.2′(判定射)** `𝒢` に対し証拠携行判定 `gateEv_i : GateEv(P_i)` を次の再帰で定め、表示用判定射をその消去像とする。

```text
gateEv_i := if h : ∃j, j ≺ i ∧ eraseEvidence(gateEv_j) ∈ {fail, na}
            then na(d := h)                    (依存失効証人 h を携行)
            else raw_i                          (RawEv ↪ GateEv の包含)
gate_i   := eraseEvidence(gateEv_i)             (gate : Idx → Gv)
```

`[DEF]` `na` を生成できるのは伝播分岐のみであり(`raw` に `na` 構成子はない)、`fail` を生成できるのは反証項を携行する `raw` のみである。

### 24.2 整礎性と伝播定理

**定理 24.3(判定射の well-definedness)** `≺` が有限かつ非循環ならば、定義 24.2′ の再帰を満たす族 `gateEv`(したがってその消去像 `gate : Idx → Gv`)が一意に存在する。`[THM]`
> 証明: 有限非循環関係は整礎であり、`gateEv_i` の値は `≺`-先行者の値のみに依存するから、整礎再帰により存在、整礎帰納により一意(消去像の一意性は関数合成)。∎

**定理 24.4(na の推移伝播)** `j ≺⁺ i` かつ `gate_j ∈ {fail, na}` ならば `gate_i = na`。`[THM]`
> 証明: `≺⁺` の経路長に関する帰納法。長さ 1 は定義 24.2′。長さ `k+1` の経路 `j ≺ k₀ ≺⁺ i` では、定義より `gate_{k₀} = na`、帰納法の仮定を `k₀` に適用。∎

**定理 24.5(現象性成分の値域)** 任意の `𝒢` について `gate_Phenomenal ∈ {open, na}`。`[THM]`
> 証明: 定義 24.2′ の場合分け。`na` 分岐はそのまま、`raw` 分岐は定義 24.2 の型制約(`RawEv(P_Phenomenal)` は `open` 構成子のみ)により `open`。∎(この定理の内容は型制約の帰結の確認であり一行だが、`pass`・`fail` 到達不能性という §15 型の禁止の**証明可能性**を担うため、注記でなく定理として置く。)

### 24.3 判定と語彙の非混同

**メタ規律 24.6′(判定射版の読み替え禁止)** `gate_i = pass` は成分 `i` の形式的充足のみを主張し、意味論側述語(付録 X′)を含意しない。特に `gate` ベクトルの全成分が `na` でないことをもって系の意味論的地位を語ってはならない。また `unk`(証明の不在)を `fail`(反証)へ読み替えることは定義 24.1′ により型水準で不能であり、記述側でも禁止する。§15′ に対応行を追加する。`[META]`

**義務 24.6(極小模型上の判定ベクトル)** 義務 13.1a の三状態模型に付随する証書に標準依存図式を適用し、判定ベクトル

```text
gate = (Core ↦ pass(相対無矛盾性の機械化後), Audit ↦ pass(定理 23.5 の恒等監査),
        Viability ↦ fail(帰還なし: 定理 5.4 により V を恒久離脱),
        Generative ↦ fail, Translation ↦ unk, Phenomenal ↦ open または na)
```

を、依存図式の選択(`Viability ≺ Phenomenal` を含むか否か)による `Phenomenal` 値の分岐込みで機械検査せよ。あわせて `M_recur` 系(義務 23.4)で `Viability ↦ pass` 側の非退化証人を与えよ。`[OBL]`
> 注意: 本義務の判定値の割当てはドラフト X4.2 のメタ命題(自己維持が `fail` の系では現象性は `na`)を、公理でなく依存図式の**選択**+定理 24.4 の帰結として実現する例示である。割当て自体は証書に依存するデータであり、対象理論の定理ではない。

## §25. 二重の非到達性 — 縦の壁と横の壁(ドラフト §X5)

**ステップ 1(直観)** 構造的充足から外部述語への含意は導出されない(縦・既証)。それとは独立に、外部述語に関与する構造をすべて保存する自己同型が二つの中心点を入れ替えるならば、いかなる構造的述語も両者を識別できない(横)。二つの壁は論理的地位が異なり、同一の確信度で語ってはならない。

### 25.1 縦の壁(再掲・新規内容なし)

**再掲 25.0** `AX_stable` は文 `∀s ∈ V(𝔽), ∀e, 𝐕_{ω(s)}(e) > 0 → P(e)`(`P` は無解釈述語記号)を導出しない(定理 7.3。`AX_core` 版は ‡)。本節はこれを変更せず、参照のみ行う。

### 25.2 シグネチャ: 不変述語族と両立自己同型

**定義 25.1(不変述語族)** 強度パラメータ

```text
τ ∈ { stat𝔎(𝔎at 静的), stat𝔽(静的 𝔽), dyn(動的 𝔽), dyn+obs(観測両立),
      dyn+lab(作動ラベル両立), dyn+rep(表現両立) }
```

に対し、**強度 τ の不変述語族**とは、各動的認証枠 `𝔻`(必要な強度では観測 `o`・ラベル付き遷移 `→ₘ`・表現 `Rep` を付随させた拡張対象)に述語 `Pred_𝔻 : A_𝔻 × S_𝔻 → Prop` を割り当て、強度 τ の任意の同型 `h : 𝔻 ≅_τ 𝔻'` について

```text
∀m s, Pred_𝔻(m, s) ⇔ Pred_𝔻'(h_A(m), h_S(s))
```

を満たすものをいう。担体成分は既定で `A` とするが、`E` や `C` を担体とする族も同形に定める(不変性の式の `h_A` を `h_E, h_C` に置換。定理 25.4-7 は `E` 担体の例)。`[DEF]`(Lean 化は「割当て+不変性証明」を束ねる宇宙多相構造体。§F ステップ 5 参照。)

**注意 25.1′(新圏の非導入・8.5 非依存)** 定義 25.1 と特殊化 25.3 は、同型 **1 本ごと**の不変性のみを用い、同型の合成・恒等を束ねた圏(群亜)を導入も要求もしない。特に設計命題 8.5 の `𝔽at`(`[CNJ]`)を実体化・変更せず、これに依存しない。強度 τ の同型たちが群亜をなすこと(両立フィールドの合成・逆射閉性)は本追補の主張に不要であり、必要になった段で別途 `[OBL]` として立てる。`[META]`

**定義 25.2(両立自己同型と中心点対称)** 強度 τ の同型は `h : 𝔻 ≅_τ 𝔻'` と書く(同型データと両立フィールドの束であり、単なる射 `𝔻 → 𝔻'` と区別する。`≈`(定義 2.4)は 𝔎at 水準の構造同型、`≅_τ` はその 𝔽-同型拡張に強度 τ の両立フィールドを束ねた表記。`≅` は v5.1 で未使用のため衝突しない)。動的 `𝔽`-同型(定義 12.1)である自己同型 `φ : 𝔻 ≅_τ 𝔻` に対し、追加両立フィールドを次で定める。

```text
obs 両立 : o ∘ φ_S = o                                  (定義 12.1 既出)
lab 両立 : ∀m s t, s →ₘ t ⇔ φ_S(s) →_{φ_A(m)} φ_S(t)、 o_ι ∘ φ_S = o_ι
rep 両立 : ∃U ユニタリ, U ∘ r = r ∘ φ_A ∧ U ∘ L = L ∘ U
```

各両立は `[FLD]`。ただし lab 両立は定義 9.4‡ の `→ₘ`・`o_ι` に依存するため `‡` を継承する(規律 0.2-6。定理 25.4-4/6・系 25.5 の `‡` はここに由来する)。rep 両立の `U` は定理 12.4 の `U_A` に対応するユニタリであり、§11.2 の全体型 `U` とは別物である。`φ` が `m ≠ m'` について `φ_A(m) = m'` を満たすとき、`φ` を `(m, m')` の**中心点対称**と呼ぶ。さらに `φ_S(s) = s` のとき **`s` 固定の中心点対称**と呼ぶ。`[DEF]`

### 25.3 横の識別不能性

**特殊化 25.3(識別不能性)** `Pred` を強度 τ の不変述語族、`φ` を強度 τ 以上の `(m, m')` 中心点対称とする。このとき `∀s, Pred(m, s) ⇔ Pred(m', φ_S(s))`。特に `φ` が `s` 固定なら `Pred(m, s) ⇔ Pred(m', s)`。
> これは定義 25.1 の不変性フィールドへの `h := φ`, `𝔻' := 𝔻` の代入(Lean では `Pred.invariant φ` の適用一行)であり、規律 0.2-1(射影非定理則)により定理を名乗らない(注意 5.3‴・定義 17.7 の「射影である」注記と同じ扱い)。本節の定理としての実質は、v5.1 の具体述語が実際に不変族をなすこと(定理 25.4)の側にある。生成元不変性から合成自己同型への不変性を**導出**する強い版は本追補の用途に不要であり導入しない(注意 25.1′ と同一の節約)。

**定理 25.4(v5.1 述語の不変族性)** 次の各述語は、示した強度の不変述語族をなす。

1. `(m, s) ↦ m ∈ H_{ω(s)}(κ(s), ε(s))` — 強度 stat𝔽。`[THM]`
2. `FM1(m, s)` — 強度 stat𝔽。`[THM]`
3. `FM2(m, s)` — 強度 stat𝔽。`[THM]`
4. `FM3(m, s)` — 強度 dyn+lab。`[THM]‡`(定義 9.4 の ‡ を継承)
5. `FM4_η(m)` — 強度 dyn+rep(状態非依存)。`[THM]`
6. `Conscious(m, s)`, `Blind(m, s)` — 強度 dyn+lab+rep(全両立)。`[THM]‡`
7. `(e, s) ↦ 𝐕_{ω(s)}(e) > 0`(E 側)— 強度 stat𝔽。`[THM]`

> 証明骨子: (1) 定理 12.2 の `h_A⟦H_w(K,X)⟧ = H'_w(h_C⟦K⟧, h_E⟦X⟧)` と静的 𝔽-同型の `κ, ε, ω` 両立、成分全単射による所属の双方向移送。(2) 全単射のもとで `ρ★(Y) ∖ {m}` の像は `ρ'★(h_C⟦Y⟧) ∖ {h_A(m)}` に等しく(単射による差集合の像の分配)、定理 12.2 と同じ計算で `h_C⟦Φ⁻ᵐ_w(Y)⟧ = Φ'⁻^{h_A(m)}_w(h_C⟦Y⟧)`。よって包含 `κ ⊆ Φ⁻ᵐ(κ)` の成否が移送される。(3) `h_E` の全単射性により `|α(m)| = |α'(h_A(m))|`(定理 2.5 の双条件 square)。(4) lab 両立により `∃s', s →ₘ s' ∧ o_ι(s') ≠ o_ι(s)` の各連言肢が移送される。(5) rep 両立の `U` はユニタリで `L` と可換なので固有空間を保ち(定理 12.4 の `U_A := U`, `T' := T` 特殊化)、`P_η ∘ U = U ∘ P_η`。よって `P_η(r(φ_A m)) = U(P_η(r(m)))`、非零性は保存。(6) (1)–(5) の連言。(7) 定理 12.2 の `𝐕` 保存の射影。∎(条件明示則: 各項の要求強度は定理文に明示した。静的同型のみで (4)(6) を主張することは偽であり、定理 12.5 の注意と同型の禁止である。)

**系 25.5(横の壁)** `𝔻` が `s` 固定の全両立中心点対称 `φ`(`φ_A(m) = m'`)を持つならば、定理 25.4 に列挙したすべての述語 — したがってそれらから論理結合で組み立てられる任意の述語 — について `Pred(m, s) ⇔ Pred(m', s)`。特に `Conscious(m, s) ⇔ Conscious(m', s)` であり、`m` と `m'` は v5.1 の述語装置の内部では識別されない。`[THM]‡`(FM3 経由の ‡ を継承)
> 証明: 特殊化 25.3 と定理 25.4。論理結合の閉性は不変性の連言・否定・量化(`A, S` 上)保存による。∎

**義務 25.6(対称二重化模型)** 次の有限模型を構成し、非自明な `s₀` 固定の全両立中心点対称の存在を機械検査せよ: `A = {m, m'}, E = {e, e'}, C = {c, c'}, S = {s₀, s₁, s₂}`。階数 0 で `α(m) = {e}, α(m') = {e'}, π(m) = {c}, π(m') = {c'}`、`σ, ρ` は逆関係(hConv・hConvP)、最上階は全関係空(Sig-2)。`κ(s₀) = {c, c'}, ε(s₀) = {e, e'}, D = {c, c'}, ω(s₀) = 0`、動態は義務 13.1a と同型の三段崩壊列。交換 `φ := (m ↔ m', e ↔ e', c ↔ c', s 恒等)` が動的 𝔽-同型かつ obs・lab 両立(観測・ラベルは対称に指定)であること、`DC(𝔽, s₀)` と `H₀ = {m, m'}` を検査せよ。rep 両立は `Ā = ℝ²` の座標交換ユニタリで与えよ。あわせて義務 22.5(実現多重性)の第二証人を兼ねること(`|A| = 2` により極小模型と非同型)。`[OBL]`
> 注意: 本模型の `α` は一点値であり `GapUp` の証人ではない。義務 23.4 の多価化と組み合わせる場合は対称性の保持を別途検査する。

### 25.4 二つの壁の地位差と束ね上げの禁止

**メタ規律 25.7(強度の異なる二種の非到達性)** 縦の壁(再掲 25.0)は**無解釈述語に対する非導出**のメタ定理(反模型存在文として機械化)であり、横の壁(系 25.5)は**不変性**の定理(自己同型の存在を前提するスキーム)である。両者は論理的型が異なる: 前者は「構造→P の橋が公理群から出ない」こと、後者は「対称性がある限り構造内述語が中心点を選べない」ことを言う。前者は自己同型の存在を要さず、後者は `P` に言及しない。一方から他方は導出されない(独立性)。両者を同一の確信度・同一の文で語ることを禁止する。`[META]`

**メタ規律 25.8(「すべての構造的述語」のサイズ規律)** 系 25.5 の「任意の述語」は、定義 25.1 の不変述語族スキーマのインスタンス(v5.1 の具体述語と、その論理結合)に対する主張である。「およそ定義可能な一切の述語」への内部量化は行わない(§F ステップ 5)。「圏論的に定義可能な性質は同型で不変である」という一般原理は、本仕様ではスキーマ+定理 25.4 の逐次検証として実現され、それを超える部分はメタ帰結として文書側に留める(定理 7.3 Lean 化注記と同一方式)。`[META]`

**帰結(束ね上げの分解)** ドラフト X5.3(日常的自己帰属は二つの独立した飛躍を束ねる)は、本節により次へ分解された: 飛躍 1(縦)は定理 7.3 が阻却(既証)。飛躍 2(横)は、系 25.5 の前提(中心点対称の存在)が成り立つ模型クラスにおいて阻却(本追補で定理化)。ただし「現実の系が当該対称性を持つ」ことは経験的仮定であり対象理論外である。束ね上げそれ自体についての主張(X5.3 の文全体)は意味論側の解釈命題として `[CNJ]` に留まる。

## §26. 語彙-命題トレーサビリティ(ドラフト §X6)

**ステップ 1(直観)** 理論に基づく自然言語記述の各文は、対応する定義・定理番号への係留を持つか、地位(`[CNJ]`・比喩)を明示しなければならない。語の含意強度は同義語代入で検定できる。

### 26.1 係留付き言明

```text

```

とする。`[DEF]`(本定義は記述の監査単位を型付けするのみであり、対象理論の命題ではない。)

**メタ規律 26.2(係留規律)** 公表される記述(ステートメント・解説・評価・判定ベクトルの読み下し)は係留付き言明の列として構成し、`anchor` を欠く断定は除去するか `cnj / metaphor` を付す。判定射(§24)に基づく記述では、各成分の値と `raw` の根拠証書への参照を `anchor` に含める。これは §15 と付録 A の隔離規律の、記述側への延長である(ドラフト X6.1 の確定)。`[META]`

### 26.2 二構成テストの翻訳論的実現

**メタ規律 26.3(語彙強度検定)** §20.1 の翻訳 `Tr` が与えられているとき、記述中の語 `v` が**証明係留的**であるとは、`v` を最強含意の同義語 `v⁺` と最中立の同義語 `v⁰` に置換した二つの記述の `Tr` による形式文が一致することをいう。一致しない場合、その語は `Tr` の像(=証明が支える内容)を超える仕事をしており、当該記述は `bridge` または `untranslated`(定義 20.1)に分類し直す。本検定は翻訳の充足保存性検査の記述側対応物である。決定可能性は無条件ではなく、次の三点を固定した場合に限る: (i) 同義語対 `(v⁺, v⁰)` は批准済みの**有限**対として与える(「最強含意」「最中立」は自然言語上一意に定まらないため、対の選定はデータであり批准対象)、(ii) `Tr` は計算可能、(iii) 形式文の比較は**構文的正規形の等値**とする(論理的同値による比較は一般に決定不能)。この三条件のもとで検定は有限の置換・比較となり CI に載る(§20.3 の有限検査に整合)。`[META]`
