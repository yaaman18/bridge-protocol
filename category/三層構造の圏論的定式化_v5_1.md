## §0. 目的と記述規律

本文書は一次仕様である。機械証明核(Lean)は本文書を照合元としてこれから書かれる。対象領域の固有名・物質名・形態名・機能名は本文の意味論語彙として用いない(意味論側との対応は付録 A に隔離する)。

### 0.1 タグ体系

- `[DEF]` — 定義。
- `[THM]` — 本文書内に証明(または再構成可能な証明骨子)を伴う命題。Lean 化義務クラス A。
- `[FLD]` — 構造体のフィールド(データまたは仮定)。決して証明と数えない。
- `[OBL]` — 型は確定、証明は Lean 側義務。証明方針を必ず併記。
- `[CNJ]` — 真偽または定式化の適否が未確定の設計命題。
- `[OUT]` — 本版の範囲外。理由を併記。
- `[META]` — 適用範囲・信頼境界・決定可能性に関するメタ規律。対象理論の命題ではない。
- `‡` — v4 破棄に伴う再構成。原定義との一致は確認不能であり、批准待ち。

### 0.2 分類規律

1. **射影非定理則** — フィールドの取り出しと定義の展開は `[THM]` を名乗らない。
2. **非昇格則** — `[FLD]` を `[THM]` に読み替えない。
3. **排他則** — タグは一意。`[OBL]` と `[CNJ]` は排他。
4. **居住証明則** — すべての構造体定義は非退化な充足例を `[OBL]` として伴う。
5. **条件明示則** — 定理の結論がフィールドに依存する場合、依存フィールドを定理文中に明示する。
6. **再構成隔離則(新設)** — `‡` 付き定義に依存する定理・義務は `‡` を継承する。`‡` 定義の差し替えは無印の定理層に影響してはならない(依存の一方向性)。批准または差し替えにより `‡` は除去される。

### 0.3 公理パック(査読2新設・査読3で二分割)

本書の大域公理を、`‡` を含まない**安定核**と、再構成公理からなる**動的パック**に分ける。査読3の指摘通り、単一パックに `‡` 公理を混ぜると、それを基準系に述べる定理がすべて規律 0.2-6 により `‡` を継承してしまい、付録 B の影響範囲宣言とも矛盾するためである:


導出不能性の定理は原則 `AX_stable` を基準に無印で述べ、`AX_core` への拡張は `‡` を継承する系として併記する(定理 7.3 がこの形式の範例)。旧 6.3(層間整合)は未式化のためいずれにも含めない(界面宣言 6.3。凍結文書 §6-2 により 6.3 は論理的に不活性であり、除外は導出関係を変えない)。凍結文書の `AX_core` は本節の `AX_core` に読み替える。

## §1. 関係核と随伴

### 1.1 四関係と直像

集合 `A, E, C, S` と関係値写像

```text
α : A → Set(E)          σ : E → Set(A)
π : A → Set(C)          ρ : C → Set(A)
```

を置く。`[DEF]` 存在的直像を

```text
α★(N) := ⋃{α(a) | a∈N}     σ★(X) := ⋃{σ(e) | e∈X}
π★(N) := ⋃{π(a) | a∈N}     ρ★(Y) := ⋃{ρ(c) | c∈Y}
```

と定める。`[DEF]`

**定理 1.1(単調性・和の保存)** 四直像とその任意の合成は包含順序に関して単調であり、各直像は任意の和を保存する。`[THM]`
> 証明: 和の各項の単調性と結合性。∎

### 1.2 普遍持ち上げと随伴

```text
σ∀★(X) := {a∈A | α(a) ⊆ X}
```

`[DEF]`

**定理 1.2(Galois 接続)** `α★(N)⊆X ⇔ N⊆σ∀★(X)`。順序圏の関手として `α★ ⊣ σ∀★`、単位 `N⊆σ∀★(α★(N))` と余単位 `α★(σ∀★(X))⊆X` が成り立つ。`[THM]`
> 証明: 両辺とも `∀a∈N, α(a)⊆X`。∎

### 1.3 剛性定理(旧公理の不可能性)

**定理 1.3(剛性)** 独立に与えた `σ` の存在的直像 `σ★` が `∀N X, α★(N)⊆X ⇔ N⊆σ★(X)` を満たすことと、`α` が一点値(実質関数 `f`)かつ `σ(e)=f⁻¹({e})` であることは同値。`[THM]`
> 証明: `N={a}, X=∅` で `α(a)≠∅`。`|α(a)|≥2` なら各単集合 `X={e}` で `a∉σ(e)`、しかし `X=α(a)` で右辺が偽となり矛盾。残りは前像の計算。∎

**帰結** 存在的直像(和保存)は退化する場合を除き右随伴(積保存)になれない。原意味論文書 Def 2.2 が像作用素 `σ★` を右側に置くのはこの型の誤りであり、意図(「行為と知覚は同一構造の二面」)は次の双対整合で定式化する。

### 1.4 双対整合系

```text
𝕂 := (A, E, C, S, α, σ, π, ρ, hConv, hConvP)
hConv  : ∀a e, e∈α(a) ⇔ a∈σ(e)
hConvP : ∀a c, c∈π(a) ⇔ a∈ρ(c)
```

構造は `[DEF]`、`hConv, hConvP` は `[FLD]`。

**定理 1.4(整合下の随伴)** `hConv` のもとで、`σ` から作る普遍持ち上げ `σ∀(X) := {a | ∀e, a∈σ(e)→e∈X}` は `σ∀★` に等しく、したがって `α★ ⊣ σ∀`。二つの右側候補の等式は定理である。`[THM]`
> 証明: `hConv` より `{e | a∈σ(e)} = α(a)`。∎

**注意 1.5(役割の分離)** `σ★`(存在的)は随伴の右側ではなく、§3 の `Θ` と §4 の蝶番 `H` の構成素である。`σ∀`(普遍的)が随伴の右側である。

**義務 1.6** `|A|=|E|=2` で `α` が真に多価な `𝕂` の充足例。`[OBL]`

## §2. 核の圏・構造同型・階数付け

### 2.1 圏 `𝔎at` と構造同型

**定義 2.1(圏 `𝔎at`)** 対象は `𝕂`。射 `h : 𝕂 → 𝕂'` は写像四つ組 `(h_A, h_E, h_C, h_S)` で lax square

```text
e∈α(a) → h_E(e) ∈ α'(h_A(a))
c∈π(a) → h_C(c) ∈ π'(h_A(a))
```

を満たすもの。恒等・合成は成分ごと。`[DEF]`

**定理 2.2** `hConv, hConvP` のもとで、`α` の lax square は `σ` の lax square を、`π` のそれは `ρ` のそれを自動的に含意する。`[THM]`
> 証明: `a∈σ(e) ⇔ e∈α(a) → h_E(e)∈α'(h_A(a)) ⇔ h_A(a)∈σ'(h_E(e))`。∎

**定理 2.3** `𝔎at` は圏である。`[THM]`(含意の連鎖と成分合成)

**定義 2.4(構造同型)** `𝕂 ≈ 𝕂'` :⇔ `𝔎at` における同型。`[DEF]`

**定理 2.5(同型の特徴づけ)** `h` が同型 ⇔ 四成分が全単射で square が双条件に強まる: `e∈α(a) ⇔ h_E(e)∈α'(h_A(a))`(他も同様)。`[THM]`
> 証明: 逆射の lax square が逆向き含意を与える。逆に双条件つき全単射の逆写像は lax。∎

### 2.2 階数基底

**定義 2.6(階数圏)** `𝒲` は有限線形順序 `(W, ≤)` に最小元 `0` と最大元 `⊤` を備えた薄い圏。`0 ≠ ⊤`。`[DEF]`

**定義 2.7(関係前層系)** 反変関手 `ℛ : 𝒲ᵒᵖ → 𝔎at`。`[DEF]`

**定義 2.8(定台前層系)** 台 `(A,E,C,S)` を固定し、各成分写像が恒等である関係前層系。すなわち階数付き関係族 `α_w, σ_w, π_w, ρ_w`(各 `w` で `hConv, hConvP`)であって、制限射が恒等四つ組であるもの。`[DEF]`

**定理 2.9(定台前層性 = 反単調族)** 定台の族が関係前層系をなすこと(Sig-1)と、関係族が階数に関して反単調であること

```text
u ≤ v → α_v(a) ⊆ α_u(a)   (σ, π, ρ も同様)
```

は同値。`[THM]`
> 証明: 恒等成分の lax square `e∈α_v(a) → e∈α_u(a)` はまさに包含。関手法則は恒等射で自明。∎

以下、`𝕂_w := ℛ(w)` と書き、直像・作用素に添字 `w` を付す。原意味論文書の「四前層性」検証点は定理 2.9 の反単調性検査に帰着する。

## §3. 余帰納核(階数付き)

### 3.1 Knaster–Tarski 構成

```text
Φ_w(Y) := π_w★(ρ_w★(Y)) : Set(C) → Set(C)
Θ_w(X) := α_w★(σ_w★(X)) : Set(E) → Set(E)
Ψ_w(Y) := Y ∩ Φ_w(Y)         Ψᴱ_w(X) := X ∩ Θ_w(X)
```

`[DEF]`

**定理 3.1** `Φ_w, Θ_w` は単調。`[THM]`(定理 1.1 の系)

```text
νΦ_w := ⋃{Y ⊆ C | Y ⊆ Φ_w(Y)}        νΘ_w := ⋃{X ⊆ E | X ⊆ Θ_w(X)}
```

`[DEF]`

**定理 3.2(Knaster–Tarski)** 各 `w` について: (i) 余帰納原理 `Y⊆Φ_w(Y) → Y⊆νΦ_w`、(ii) `Φ_w(νΦ_w) = νΦ_w`、(iii) `νΦ_w` は後不動点の最大元。`Θ_w` も同様。`[THM]`
> 証明: 後不動点の和が後不動点(単調性)、単調性で不動点に閉じる。∎

**定理 3.3(終余代数性)** 薄い圏 `(Set(C),⊆)` 上、`Φ_w`-余代数(後不動点)と包含射のなす圏で `(νΦ_w, =)` は終対象。`[THM]`

### 3.2 台 sieve と臨界階数

```text
Sieve_Φ(Y) := {w ∈ W | Y ⊆ Φ_w(Y)}
```

`[DEF]`

**定理 3.4(sieve 性)** 定台前層系(定理 2.9 の反単調性)のもとで `Sieve_Φ(Y)` は下方閉、すなわち `𝒲` 上の sieve である。`[THM]`
> 証明: `u≤v, Y⊆Φ_v(Y)` なら反単調性と単調性で `Φ_v(Y) ⊆ Φ_u(Y)`。∎

**定義 3.5(臨界階数)** `Sieve_Φ(Y) ≠ ∅` のとき `w_crit(Y) := max Sieve_Φ(Y)`。`[DEF]`

**定理 3.6(存在)** `W` は有限線形順序なので、非空 sieve は最大元を持ち `w_crit` は well-defined。また `Sieve_Φ(Y) = ↓w_crit(Y)`。`[THM]`

**公理 3.7(Sig-2)** ある `w* < ⊤` が存在し、すべての非空 `Y ⊆ C` について `Sieve_Φ(Y) ⊆ ↓w*`(同値に `w_crit(Y) ≤ w*`)。`[FLD]`

**定理 3.8** Sig-2 のもとで、`w > w*` なるすべての `w` について `νΦ_w = ∅`。`[THM]`
> 証明: 非空後不動点 `Y` があれば `w ∈ Sieve_Φ(Y) ⊆ ↓w*` で矛盾。∎

## §4. 認証構造と蝶番

### 4.1 認証枠・認証述語・認証構造(査読1指摘による三分離)

```text
H_w(K, X) := ρ_w★(K) ∩ σ_w★(X) ⊆ A
```

`[DEF]`(意味論側呼称: 蝶番作動 `Act_w`。付録 A)

まず状態を選ばないデータを**認証枠**として分離する:

```text
𝔽 := (ℛ, κ, ε, D, ω)
κ : S → Set(C)    ε : S → Set(E)    D ⊆ C    ω : S → W
```

`[DEF]` 次に各状態を評価する**認証述語**を定める:

```text
h₁(𝔽,s) :⇔ κ(s) ⊆ Φ_{ω(s)}(κ(s))        h₂(𝔽,s) :⇔ ε(s) ⊆ Θ_{ω(s)}(ε(s))
h₃(𝔽,s) :⇔ H_{ω(s)}(κ(s), ε(s)) ≠ ∅      h₄(𝔽,s) :⇔ κ(s) ∩ D ≠ ∅
DC(𝔽,s) :⇔ h₁ ∧ h₂ ∧ h₃ ∧ h₄            V(𝔽) := {s ∈ S | DC(𝔽,s)}
```

`[DEF]` 特定状態の認証**証拠**が必要な文脈では、証明携行対

```text
Δ := (𝔽, s, dc : DC(𝔽,s))
```

を用いる。`[DEF]`(`dc` は `[FLD]`。任意状態への証明は `Δ` に含まれず、`V(𝔽)` の評価は述語 `DC` で行う — 旧版は特定 `s` の証明フィールドと全状態量化 `V` を同一構造に混在させていた。以降、文脈から `𝔽` が明らかな場合 `DC(s)`, `V` と略記するが、常に本節の述語形を指す。)

**定理 4.1(余帰納的正規化)** `h₁(𝔽,s)` ならば `κ(s) ⊆ νΦ_{ω(s)}`、`h₂(𝔽,s)` ならば `ε(s) ⊆ νΘ_{ω(s)}`。`[THM]`(定理 3.2(i))

**定理 4.2(空性伝播・分離不能性)** `κ(s)=∅` または `ε(s)=∅` ならば `H_{ω(s)}(κ(s),ε(s))=∅`。したがって `h₃` は両配置の非空を強制し、片側のみの配置は認証不能。`[THM]`
> 証明: `ρ★(∅)=σ★(∅)=∅`、交差は空。∎

**定理 4.3(認証の臨界拘束)** Sig-2 のもとで、`h₁(𝔽,s) ∧ h₄(𝔽,s)` ならば `ω(s) ≤ w* < ⊤`。認証は臨界階数以下でのみ可能であり、最上階での認証は存在しない。`[THM]`
> 証明: `h₄` より `κ(s)≠∅`、`h₁` より `ω(s) ∈ Sieve_Φ(κ(s)) ⊆ ↓w*`。∎

**義務 4.4** 義務 1.6 の族上に、`V(𝔽) ≠ ∅` なる認証枠 `𝔽` の充足例。`[OBL]`

## §5. 階付き動態・崩壊・到達構造

### 5.1 動的認証枠と階付き動態(査読3: 動態の型を確定)

**定義 5.0(動的認証枠)** 認証枠 `𝔽`(§4.1)に動態データを添えた組

```text
𝔻 := (𝔽, δ, Trans_i, Trans_e, h_int)
δ : W × Set(C) → W                    Trans_i, Trans_e ⊆ S × S
h_int : ∀(s,t) ∈ Trans_i,
        κ(t) = Ψ_{ω(s)}(κ(s)) ∧ ε(t) = Ψᴱ_{ω(s)}(ε(s)) ∧ ω(t) = δ(ω(s), κ(s))
```

を**動的認証枠**と呼ぶ。`h_int` は `[FLD]`(内的遷移が更新則を実現するという整合条件)。記法: `s →ᵢ t :⇔ (s,t) ∈ Trans_i`、`s →ₑ t :⇔ (s,t) ∈ Trans_e`、`→ := →ᵢ ∪ →ₑ`。タグ付き辺の型 `Edge` と余積分解は §11.1。`[DEF]`

**定義 5.1(構成空間と全域更新写像)** 構成空間を `Conf := Set(C) × Set(E) × W` とし

```text
upd(K, X, w) := ( Ψ_w(K), Ψᴱ_w(X), δ(w, K) )        (Ψ_w(K) = K ∩ Φ_w(K)、Ψᴱ_w(X) = X ∩ Θ_w(X))
```

を**全域**写像 `upd : Conf → Conf` として定める。状態の構成を `conf(s) := (κ(s), ε(s), ω(s))` と書けば、`h_int` は `s →ᵢ t → conf(t) = upd(conf(s))` と言い換えられる。`[DEF]`(査読3: 旧版は更新式 `κ′,ε′,ω′` を状態間関係に接続しておらず、更新後状態の存在も未仮定だった。`upd` は構成空間上の全域関数なので存在仮定を要さず、状態層とは `h_int` で結ばれる。)

**定理 5.2(内的降下)** `upd` の軌道に沿って κ 成分は包含で単調非増加。さらに `K ⊄ Φ_w(K)` の段では厳密に減少する。`[THM]`
> 証明: `K ∩ Φ_w(K) ⊆ K`、非後不動点なら不等。∎

### 5.2 有限歩崩壊(再構成前提つき)

**公理 5.3(R2′)‡** `𝔻` の漂移 `δ` は非減少 `w ≤ δ(w,K)` であり、`K ≠ ∅ ∧ w < ⊤` のとき厳密に増加 `w < δ(w,K)`。`[FLD]‡`

```text
Ô_w[m,e] := T_w[m,e] · 𝐕_w(e)
```

`[DEF]`(v5 Def 4.5 の `𝐓 ⊙ 𝐕`)

## §8. 橋渡し(離散層 ↔ 連続層)

### 8.1 無条件含意の否定

**定理 8.1(前向反例)** `DC(𝔽,s)` の任意の充足例に対し `T := 0` と置けば `L = 0`、`Fix(L) = {0}` で `¬Nontriv(L)`。連結データなしの `DC → Nontriv(L)` は偽。`[THM]`

**定理 8.2(逆向反例)** `T := id` なら `Nontriv(L)`。一方 `D := ∅` の下で `h₄(𝔽,s)` は全状態で偽となり `V(𝔽) = ∅`。`Nontriv(L) → ∃s, DC(𝔽,s)` は偽。`[THM]`

### 8.2 表現構造と輸送補題

```text

r : A → Ā
h_ne  : ∀u ∈ H_{ω(s)}(κ(s),ε(s)), r(u) ≠ 0
h_fix : ∀u ∈ H_{ω(s)}(κ(s),ε(s)), L(r(u)) = r(u)

IRep := (𝔽, s, Ā, L, r, δᴬ, h_comm, h_id, h_ne)
h_comm : ∀u, L(r(u)) = r(δᴬ(u))
h_id   : ∀u ∈ H_{ω(s)}(κ(s),ε(s)), δᴬ(u) = u
```

`[DEF]`(証明フィールドは `[FLD]`)

**定理 8.3(前向輸送)** `Rep` の `h_ne, h_fix` と **`h₃(𝔽,s)`** から `Nontriv(L)`、さらに `r(H) ⊆ Wld₀(L)∖{0}` が従う。`IRep` からは `h_comm, h_id, h_ne` と `h₃(𝔽,s)` により同結論。`[THM]`
> 証明: `h₃` で `u∈H` を取る。`r(u)≠0` かつ固定、`Wld₀ = Fix`(定理 6.6-2)。∎
> `h₃` なしでは `H=∅` によりフィールドが空虚に真となり結論は出ない(規律 0.2-5)。

**注意 8.4(誠実性)** `h_fix`(`h_id`)は結論を `r` に沿って引き戻した形の仮定であり、定理 8.3 は輸送補題であって生成定理ではない。無条件生成は定理 8.1 により不可能なので、これは欠陥でなく橋の正確な強さである。v5 追記の `WorldDC.lean` 弱形式(「DC と非零固定方向が与えられたとき Act 非空かつ WldNontrivial」)は本定理に一致する。

### 8.3 v5 Prop 5.3 の型修理と現状

v5 Prop 5.3(`DC ⟺ 𝐖𝐥𝐝` 非自明かつ `Act` 参与)の原証明は、離散集合 `ε ⊆ T'_w(ε)` を連続作用素 `𝐋_w` の安定部分空間への所属と「同値」とするが、両辺の型が異なり(集合 対 部分空間)、この同値は連結写像なしには文にならない。修理後の正確な形は:

**設計命題 8.5** 次を満たす関手対の存在。`[CNJ]`


**定理 8.6(Umwelt 相対性・可証部分)** `Wld_η(L)` は `T` のみの関数であり(定理 6.7)、`T ≠ T'` で `Wld` は実際に異なる場合がある: `m ≥ 1` かつ `0 ≤ η < 1` のとき `Wld_η(0) = {0} ≠ Ā = Wld_η(id)`。`η ≥ 1` では両者とも全空間となり区別されない(査読3: 区別の成立範囲を明示)。`[THM]`

### 10.1 見果てぬ set point の圏論化

**注意 10.3** §6 の帯域中心 `1` は `Fix(L)` の固有値という数値基準であり、圏内の終対象ではない(v5 追記の解釈注意)。`NoT*` ガードと矛盾しない。

### 10.2 保護トレースと非干渉

```text
Trace := (externalToken : Option(X), core : N, val : Val,
          R : J → J → Prop, sequence : List(A))
M₄a(t) :⇔ t.externalToken = none        M₄b(t) :⇔ NoT*(t.R)
M₄(t) :⇔ M₄a(t) ∧ M₄b(t)
observe : Selector → Individual → Trace
Pure(observe) :⇔ ∀s₁ s₂ i, observe(s₁,i) = observe(s₂,i)
Sys(変異の担体)        prot : Sys → Trace        (保護射影)
Mutation := Sys → Sys
TraceSafe(μ : Mutation) :⇔ ∀x, prot(μ(x)) = prot(x)
M₄Safe(μ : Mutation)   :⇔ ∀x, M₄(prot(x)) → M₄(prot(μ(x)))
```

`[DEF]`(査読4: 変異の担体 `Sys` と保護射影 `prot` を分離。査読3版の `μ : Trace → Trace` では `TraceSafe` が恒等条件に潰れ、保護部分に触れない非自明な変異をすべて排除していた。本定義は「保護トレースを変えない」を射影等式として型付けし、非保護部分の変異を許す。`Sys := Trace, prot := id` と置けば旧定義は特殊例として回収される。旧フィールド名 `value : V` は生存圏 `V` と衝突するため `val : Val` に改名(査読3)。`Pure` は観測等値による非干渉性の定義そのもの。`M₄Safe` の適用は展開であり定理を名乗らない。)

**定理 10.4** ある `t'` がすべての点から到達されるよう保護トレースの `R` を書き換える変異 `μ`(`prot(μ(x)).R` が終対象を持つ)について、`M₄b(prot(μ(x)))` は偽。`[THM]`(定理 10.2)

**定理 10.5** `TraceSafe(μ) → M₄Safe(μ)`。`[THM]`
> 証明: `prot(μ(x)) = prot(x)` による書き換え。∎(査読2: 旧文の `Pure` は本定理に不要であり、結論は定義済みの `M₄Safe` で述べる。`Pure` の役割は非干渉の定義それ自体と、定理 10.6 の不十分性側にある。)

**定理 10.6(反例)** `Pure` 単独は `M₄Safe` を含意しない: 第一引数無視の `observe`(`Pure` 成立)と、定理 10.4 の終対象書き込み変異 `μ` の組が反例。`[THM]`

## §11. 余積・辺型・分解

### 11.1 依存和の余積普遍性

**定理 11.1** `Σ i, P(i)` は流入 `in_i(x) := ⟨i,x⟩` を伴う `Type` の余積: 任意の型 `Z` と族 `f_i : P(i) → Z` に余対 `[f]` が存在し、ベータ則を満たし一意(査読対応: 生存圏 `V` との記号衝突を避け終域を `Z` とする)。`[THM]`
> 証明: 依存和の帰納法と関数外延性。∎

遷移は最初から**タグ付き辺の型**で与える(Sig-3。査読1・2: 状態対集合と余積型の混用を排除):

```text
Λ := {i, e}        Trans_i, Trans_e ⊆ S × S
Edge := Σ λ:Λ, Trans_λ                    (辺 = タグと遷移対の依存対)
src⟨λ,(s,s')⟩ := s      tgt⟨λ,(s,s')⟩ := s'      tag⟨λ,(s,s')⟩ := λ
s → s' :⇔ ∃λ, (s,s') ∈ Trans_λ
```

`[DEF]` 境界は `Edge` 上の述語 `BdEdge` として定める(査読3: §5.3 の対レベル `BdPair` と同一記号を再定義していた二重定義を、別名により解消):

```text
BdEdge(x) :⇔ src(x) ∈ ↓V ∧ tgt(x) ∈ K
Bd_int := {x ∈ Edge | BdEdge(x) ∧ tag(x) = i}      Bd_ext := {x ∈ Edge | BdEdge(x) ∧ tag(x) = e}
```

`[DEF]`

**定理 11.2** (i) 標準写像 `Bd_int ⊔ Bd_ext → {x ∈ Edge | BdEdge(x)}`(部分型の余積から余積の部分型へ)は全単射である。すなわち境界辺はタグで過不足なく分割される。(ii) 対レベルとの対応: `(s,t) ∈ BdPair ⇔ ∃x ∈ Edge, BdEdge(x) ∧ src(x) = s ∧ tgt(x) = t`。(iii) `tag(x) = e ∧ BdEdge(x)` なる辺で端点の核が `≈` を満たすものは移送辺である(定義 5.5)。`[THM]`
> 証明: (i) 単射はタグ(依存対の第一成分)の排他性、全射は場合分け。(ii) 両定義の展開と `→` の存在量化。∎(査読2: 旧文 `Bd = Bd_int ⊔ Bd_ext` は集合と余積型の間の型不整合な等号であった。)

### 11.2 標準比較と分解可能性

索引型 `I`、依存族 `P : I → Type`、全体型 `U`(査読2: 階数型 `W` との記号衝突を避けて改名)、流入族 `ι_i : P(i) → U` に対し

```text
c := [ι] : (Σ i, P(i)) → U
Reducible(c)  := Equiv データ (c, c⁻¹, 左逆則, 右逆則)
ReducibleP(c) :⇔ Function.Bijective(c)
Irreducible(c) :⇔ ¬ReducibleP(c)
```

`[DEF]`

**定理 11.3** `Reducible → ReducibleP`。`ReducibleP ⇔ ∀ y : U, ∃! ⟨i,x⟩, c⟨i,x⟩ = y`(査読2: 型上の量化の記法を修正)。`[THM]` 逆方向はデータ構成に選択原理を要する(`Equiv.ofBijective` に隔離)。`[OBL]`

INS(§5.4)は観測代数上の分解不能性、`Irreducible` は余積比較の分解不能性であり、両者は別命題である。混同を禁止する(§15)。

## §12. 不変性定理(≈ 双模倣)

### 12.1 Δ-同型

**定義 12.1(静的・動的 `𝔽`-同型)** 集合像への持ち上げを `h⟦X⟧ := {h(x) | x ∈ X}` と書く(査読2: 旧記法 `h_C ∘ κ` は点写像と集合値写像の合成であり型不整合。以下すべて像で述べる)。核の `𝔎at`-同型 `h = (h_A, h_E, h_C, h_S)`(定義 2.4、定理 2.5 により strict square 全単射)が**静的 `𝔽`-同型**であるとは — 状態写像には `h` の第四成分 `h_S`(定義 2.1)をそのまま用い、新たな写像は添えない(査読3: 重複の解消)—

```text
∀s, h_C⟦κ(s)⟧ = κ'(h_S(s))      ∀s, h_E⟦ε(s)⟧ = ε'(h_S(s))
h_C⟦D⟧ = D'                     ∀s, ω'(h_S(s)) = ω(s)
```

を満たすこと。さらにタグごとの遷移両立

```text
∀λ ∀s t, (s,t) ∈ Trans_λ ⇔ (h_S(s), h_S(t)) ∈ Trans'_λ
```

を満たすものを**動的 `𝔽`-同型**と呼ぶ(査読1: 到達構造の保存には遷移の保存が不可欠)。観測両立版はさらに `o' ∘ h_S = o` を課す。階数族に対しては前層系の自然同型として与える。`[DEF]`

### 12.2 静的・内的保存(定理)

**定理 12.2(静的双模倣)** `𝔎at`-同型 `h` について:

```text
h_E⟦α★(N)⟧ = α'★(h_A⟦N⟧)      h_C⟦π★(N)⟧ = π'★(h_A⟦N⟧)    (σ, ρ も同様)
h_C⟦Φ_w(Y)⟧ = Φ'_w(h_C⟦Y⟧)     h_C⟦νΦ_w⟧ = νΦ'_w
h_A⟦H_w(K,X)⟧ = H'_w(h_C⟦K⟧, h_E⟦X⟧)
```

したがって静的 `𝔽`-同型は `h₁–h₄` すなわち `DC` を双方向に保存し、`Sieve_Φ, w_crit, 𝐕_w` も保存する(全単射は濃度を保つ)。`[THM]`
> 証明: strict square と全単射で `∃` 節が両方向に移る。直像の等式から `Φ` の共役、後不動点の対応、和の像で `νΦ`。交差は単射で像と交換。∎

**定理 12.3(配置成分の内的双模倣)** 静的 `𝔽`-同型のもとで、更新の κ・ε 成分は可換: `h_C⟦Ψ_{ω(s)}(κ(s))⟧ = Ψ'_{ω(s)}(h_C⟦κ(s)⟧)`(階数は `ω'(h_S(s)) = ω(s)` で一致)。`ε` 側も同様。`[THM]`(定理 12.2 と、単射による交差の像の分配)

**定義 12.3′(漂移等変)** 静的 `𝔽`-同型が**漂移等変**であるとは、`∀w K, δ'(w, h_C⟦K⟧) = δ(w, K)` を満たすことをいう。`[FLD]`

**定理 12.3″(全更新の双模倣)** 静的 `𝔽`-同型が漂移等変なら、`h_Conf(K,X,w) := (h_C⟦K⟧, h_E⟦X⟧, w)` について `h_Conf ∘ upd = upd' ∘ h_Conf`。`[THM]`
> 証明: κ・ε 成分は定理 12.3、第三成分は漂移等変そのもの。∎(査読4: 旧 12.3 は「内的更新の可換」を名乗りながら `upd` の第三成分 `δ(w,K)` の等変条件を欠いていた。名称を配置成分に限定し、全体可換は等変条件付きの本定理に分離。)

**定理 12.4(連続層の保存)** `T' = U_E ∘ T ∘ U_A⁻¹`(`U_A, U_E` ユニタリ)のとき `L' = U_A L U_A⁻¹` であり、`spec(L')=spec(L)`、`Wld_η(L') = U_A(Wld_η(L))`、`χ(L')=χ(L)`、`Nontriv` 保存。`[THM]`
> 証明: ユニタリ共役はスペクトルと固有空間を移す。∎

**定理 12.5(観測両立動的同型と INS)** 観測両立**動的** `𝔽`-同型は `V, ↓V, K` を対応させ、INS を双方向に保存する。`[THM]`
> 証明: 静的部分(定理 12.2)により `DC` が状態ごとに対応し `h_S⟦V⟧ = V'`。遷移両立により `→` と `→'` が `h_S` で一対一に対応し、反射推移閉包 `→*` も対応する。よって `h_S⟦↓V⟧ = ↓V'`、`h_S⟦K⟧ = K'`。観測両立 `o' ∘ h_S = o` により繊維が対応し、混在繊維の有無(定理 5.7)が一致する。∎
> 注意(査読1): 静的同型のみでは本定理は**偽**である。静的データと観測が一致しても、片方の系のみが遷移 `x → v` を持てば `↓V` と `K` が食い違う。動的両立が不可欠。

v5 の「テンソル `𝐓,𝐕,𝐖𝐥𝐝` も ≈ で保存」は定理 12.2(𝐕)と定理 12.4(𝐓, 𝐖𝐥𝐝)に分かれて成立する。

### 12.3 外的公理

## §17. 開放動態・回復・ライブネス

**追加規律**: 本節以降は §1–16 の既存核を変更しない追加層である。自己関手による決定的更新を primitive にせず、§5 の非決定的な `Trans_i, Trans_e` と `formal/ERIEC/OpenEvolution.lean` の集合値遷移を同じ型で受けられる、ラベル付きグラフとその自由経路圏を基礎にする。これにより、始域・終域に依存する射を単一集合 `Mor(𝒞)` に押し込める型問題と、異なる終域を持つ射の等式問題を回避する。

### 17.1 ラベル付き開放グラフと自由経路圏

**定義 17.1(辺種別・開放グラフ)** 結合口の型 `Port` に対し

```text
EdgeKind(Port) := internal | coupling(Port)

OpenGraph :=
  (State : Type,
   step  : State → EdgeKind(Port) → State → Prop)
```

を定める。`[DEF]` `internal` は指定境界内の更新、`coupling(p)` は口 `p` を通る境界横断遷移である。個体群選択・設計・学習条件設定等のメタ操作は対象内 `step` に混入させず §19 の系譜層に置く。

**定義 17.2(ラベル付き経路)** `Path_G(s,t)` を次の帰納型とする。

```text
nil  : Path_G(s,s)
cons : step_G(s,k,u) → Path_G(u,t) → Path_G(s,t)
```

経路長を `len(nil)=0`, `len(cons(e,p))=1+len(p)`、全辺が `internal` であることを `InternalPath(p)` と定める。`[DEF]`

**定理 17.3(自由経路圏)** 状態を対象、`Path_G(s,t)` を射、`nil` を恒等射、経路連結を合成とすると圏 `FreePath(G)` をなす。`internal` 辺だけで生成される射は同じ対象を持つ部分圏 `FreePath_i(G)` をなす。`[THM]`
> 証明: 左右単位則と結合則はリスト連結の帰納法。内部辺だけからなる性質は空経路と連結で閉じる。∎

### 17.2 開放枠・実行・有限内的地平

**定義 17.4(開放枠)**

```text
𝒪 := (G, Init, A, B, Fair)
Init : G.State → Prop       初期状態領域
A    : G.State → Prop       許容述語
B    : G.State → Prop       回復包絡、A ⊆ B
Fair : Execution(G) → Prop  公平性条件
```

を開放枠と呼ぶ。`A ⊆ B` は `[FLD]`、その他はデータである。`[DEF]` 公平性を要求しない用途では `Fair := True` と明示する。

**定義 17.5(無限実行)**

```text
Execution(G) :=
  (state : ℕ → G.State,
   kind  : ℕ → EdgeKind(Port),
   edge  : ∀n, G.step (state n) (kind n) (state (n+1)))

PureInternal(x) :⇔ ∀n, x.kind n = internal
```

`[DEF]` 有限区間 `n≤m` からは標準的に接頭経路 `segment(x,n,m) : Path_G(x_n,x_m)` が得られ、長さは `m-n` である。これは実行の辺を有限回連結する構成である。

**定義 17.6(有限内的地平)**

```text
FIH(G,A) :⇔
  ∀s, A(s) → ∃N,
    ∀t (p : Path_G(s,t)),
      InternalPath(p) → N ≤ len(p) → ¬A(t)
```

`[DEF]` 一つの決定的軌道でなく、`s` から出るすべての純内的経路を量化する。内的経路が無限に延長できることは FIH に含めず、必要なら

```text
InternalTotal(G) :⇔ ∀s, ∃t, G.step s internal t
```

を別フィールドとして課す。`[DEF]`

### 17.3 帰還、回復、自己維持

**定義 17.7(実行上の帰還性)**

```text
Recurrent_A(x) :⇔ ∀n, ∃m ≥ n, A(x.state m)              -- □◇A
BoundedRecurrent_A(k,x) :⇔ ∀n, ∃m, n ≤ m ≤ n+k ∧ A(x.state m)
```

`[DEF]` `BoundedRecurrent_A(k,x) → Recurrent_A(x)` は射影である。

**定義 17.8(三段階のライブネス)**

```text
PossibleLive(𝒪) :⇔
  ∃x, Init(x.state 0) ∧ Recurrent_A(x)

FairRealizable(𝒪) :⇔
  ∀s, Init(s) → ∃x, x.state 0 = s ∧ Fair(x)

FairLive(𝒪) :⇔
  FairRealizable(𝒪) ∧
  ∀x, Init(x.state 0) → Fair(x) → Recurrent_A(x)

BoundedLive(k,𝒪) :⇔
  FairRealizable(𝒪) ∧
  ∀x, Init(x.state 0) → Fair(x) → BoundedRecurrent_A(k,x)
```

`[DEF]` `FairRealizable` を含めることで、公平な実行が一本も存在しないために全称条件が空虚に成立することを防ぐ。本仕様で単に「自己維持可能」と言うときは `PossibleLive`、「公平な開放自己維持」と言うときは `FairLive` を指す。製品・実装の持続保証に `PossibleLive` だけを用いてはならない。

**定義 17.9(回復可能性と不可逆部)**

```text
Recoverable(B,A) :⇔
  ∀s, B(s) → ∃t, A(t) ∧ Nonempty(Path_G(s,t))

RecoverableWithin(k,B,A) :⇔
  ∀s, B(s) → ∃t (p : Path_G(s,t)), A(t) ∧ len(p) ≤ k

Down(A)(s) :⇔ ∃t, A(t) ∧ Nonempty(Path_G(s,t))
K_A(s)    :⇔ ¬Down(A)(s)
```

`[DEF]` `B∖A` が一時的離脱領域、`K_A` が指定動態に関する不可逆部である。

**定理 17.10(不可逆部の前方閉性)** `K_A(s)` かつ `G.step s k t` なら `K_A(t)`。`[THM]`
> 証明: `t` から `A` への経路があれば、先頭に `s→t` を連結して `s` から `A` への経路を得る。対偶。∎

これは定理 5.5 の抽象形である。従って回復公理は `K_A` からの復帰として置かず、`B ⊆ Down(A)` または `B=Down(A)` として置く。

### 17.4 有限内的地平との両立

**定理 17.11(純内部非帰還)** `FIH(G,A)` のもとで、任意の純内部実行 `x` は `Recurrent_A(x)` を満たさない。`[THM]`
> 証明: 帰還を仮定して `A(x_{m₀})` を取る。FIH の地平を `N` とし、再度の帰還から `m≥m₀+N` かつ `A(x_m)` を取る。`segment(x,m₀,m)` は長さ `m-m₀≥N` の内部経路なので FIH に反する。∎

**定理 17.12(開放帰還との両立)** FIH と `PossibleLive` は同時充足可能である。`PossibleLive` は FIH から導出されず、`FairLive` は `PossibleLive` から導出されない。`[THM]`
> 証人:
>
> 1. `M_recur`: 状態 `{a,b}`、`Init=A={a}`、`B={a,b}`、`Fair=True`、内部辺 `a→b, b→b`、結合辺 `b→a`。FIH は `N=1`、`Recoverable(B,A)` も成立する。`a,b,a,b,…` が帰還証人なので `PossibleLive`。一方 `a,b,b,b,…` も実行なので `FairLive` でない。
> 2. `M_fall`: 状態 `{a,b}`、`Init=A=B={a}`、`Fair=True`、内部辺 `a→b,b→b`、結合辺なし。FIH は同じだが帰還実行は存在しない。
> 3. `M_fair`: `M_recur` と同じグラフで、`FairRepair(x) :⇔ ∀n, x_n=b → ∃m≥n, x_m=b ∧ x.kind(m)=coupling(unit)` とする。交代実行が公平性の居住証人であり、公平な任意の実行は各 `b` 滞在からいつか `a` へ戻るため `FairLive`。さらに局所 scheduler 条件 `∀n, x_n=b → x.kind(n)=coupling(unit)` を公平性に採れば `BoundedLive(1)` の非退化証人になる。
>
> いずれも有限表の直接検査。∎

公平性は動態から自動で生じない。標準候補は「継続的に可能な指定結合辺はいつか発火する」という弱公平性、「無限回可能になる指定辺は無限回発火する」という強公平性、または scheduler の期限付き選択証明である。どれを採るかは対象実装の証書に記録する。`[FLD]`

### 17.5 到達成分上の終対象ガード

**定義 17.13(初期到達成分)**

```text
Reach_Init(t) :⇔ ∃s, Init(s) ∧ Nonempty(Path_G(s,t))

HasT*_Init(G) :⇔
  ∃t, Reach_Init(t) ∧
      ∀s, Reach_Init(s) → Nonempty(Path_G(s,t))

NoT*_Init(G) :⇔ ¬HasT*_Init(G)
```

`[DEF]` これは `FreePath(G)` の `Init` から到達可能な充満部分圏を薄圏化したときの終対象不在である。到達不能な孤立状態を追加して `NoT*` を見かけ上成立させることを禁止する。

**定義 17.14(タグ込み最終周期性・状態新規性)** 実行 `x` について

```text
EventuallyPeriodic(x) :⇔
  ∃N p, 0<p ∧ ∀n≥N,
    x.state(n+p)=x.state(n) ∧ x.kind(n+p)=x.kind(n)

Aperiodic(x) :⇔ ¬EventuallyPeriodic(x)

FreshState(x) :⇔
  ∀n, ∃m≥n, ∀k<m, x.state(m) ≠ x.state(k)
```

と定める。`[DEF]` 状態だけでなく `internal/coupling(port)` のタグを周期性に含める。同じ状態列でも内部更新と結合の配置が異なる実行を同一周期と数えない。`FreshState` は一実行内の状態新規性であって、§19 の構造生成ではない。

**定理 17.15(終対象・帰還・周期の非含意)** 次はいずれも一般には導出されない。`[THM]`

```text
NoT*_Init → PossibleLive
NoT*_Init → (∃x, Init(x₀) ∧ Aperiodic(x))
PossibleLive → NoT*_Init
PossibleLive → (∃x, Init(x₀) ∧ FreshState(x))
```

> 証人 `M_branch`: 状態 `{r,l,q}`、`Init=A=B={r}`、`Fair=True`、内部辺 `r→l,l→l,q→q`、結合辺 `r→q`。三状態はすべて `r` から到達可能で、`l` と `q` は互いに到達不能なので終対象はない。一方、すべての実行は一歩後に `l` または `q` に留まり、帰還も非周期実行も存在しない。孤立・到達不能状態は使っていない。
>
> `PossibleLive → NoT*_Init` の反例は一状態自己ループ。最後の反例は状態が二個しかない `M_recur`。∎

### 17.6 具象層への厳密な埋込み

**定義 17.16(動的認証枠の開放グラフ)** §5 の動的認証枠 `𝔻` から

```text
G_𝔻.State := S
G_𝔻.step(s,internal,t) :⇔ s →ᵢ t
G_𝔻.step(s,coupling(unit),t) :⇔ s →ₑ t
```

を定める。`Init` は監査対象ごとに指定し、`A(s):⇔DC(𝔽,s)`、`B(s):⇔s∈↓V` とする。`[DEF]`

**定理 17.17(崩壊定理の経路版)‡** 定理 5.4 の仮定下で `FIH(G_𝔻,A)` が成立し、地平は一様に `|W|+|C|` 以下で取れる。`[THM]‡`
> 証明: `G_𝔻` の内部経路は定義により定理 5.4-(2) の有限内的経路そのもの。長さが崩壊上界以上なら終点の `κ=∅`、従って `h₃` が偽で `¬DC`。∎

**注意 17.18** E5 は結合遷移の同型双模倣であり `A` への帰還を主張しない。具象層で `PossibleLive/FairLive` を得るには `Trans_e`、公平性、回復包絡に関する追加フィールドが必要である。`K=S∖↓V` からの回復は定理 5.5 と矛盾するため要求しない。

**義務 17.19(開放動態参照模型)** `M_recur`, `M_fall`, `M_fair`, `M_branch` を有限型上で構成し、定理 17.3, 17.10–17.15 を Lean 化せよ。`M_recur` は凍結文書の `M↻` と対応付け、`G_𝔻` 上で同じ `W₆` 証人を与えることを示せ。`M_fair` では `FairLive` と `BoundedLive(1)` を別々に証明し、公平性の非空性も証明せよ。`[OBL]`

## §18. 監査・来歴・内生的回復

### 18.1 開放グラフのシミュレーション圏

**定義 18.1(シミュレーション射)** `G,H` の結合口が共通の型を持つとする。射 `f:G→H` は状態写像 `f₀:G.State→H.State` と

```text
maps : ∀s k t, G.step s k t → Nonempty(Path_H(f₀(s),f₀(t)))
labels : maps が作る経路は k と同じ辺種別だけを使う
```

を持つ。`[DEF]` `labels` は内部辺を結合辺へ、または結合口を別の来歴へ偽装しないために必要である。

**定理 18.2** 開放グラフとシミュレーション射は圏 `OpenSim` をなす。`[THM]`
> 証明: 恒等写像には一辺経路、合成には像経路の連結を使う。経路写像とラベル保存は帰納法で合成に閉じる。圏の三法則は状態写像の関数外延性と証明項の proof irrelevance で Hom 構造体の等式へ持ち上げる。∎

### 18.2 実装抽象化の健全性

実装遷移系を `I`、監査模型を `M` とする。抽象化証書を

```text
AuditMap(I,M) :=
  (q : I → M in OpenSim,
   initPres : ∀s, Init_I(s) → Init_M(q(s)),
   scopeSurj : ∀m, Reach_Init_M(m) →
                 ∃i, Reach_Init_I(i) ∧ q(i)=m,
   fingerprint : 実装 artifact の同一性,
   assumptions : 有限化・観測・数値誤差・未観測遷移の仮定)
```

とする。`[DEF]`

**定理 18.3(`NoT*` の監査健全性)** `AuditMap(I,M)` があり `NoT*_Init(M)` なら `NoT*_Init(I)`。`[THM]`
> 証明: 具体側に初期到達成分の終対象 `t` があれば、抽象側の任意の初期到達状態を `scopeSurj` で具体状態から持ち上げる。具体状態から `t` への経路を `q` で写すと、抽象側の全状態から `q(t)` への経路を得て矛盾。∎

存在ライブネスはこの向きのシミュレーションだけでは逆輸送できない。抽象模型にだけ存在する偽の帰還経路を排除するため、各証人辺・経路を具体実行へ持ち上げる `PathLift`、または具体 lasso/実行そのものを証書に含める。`[META]`

### 18.3 結合口の来歴型

**定義 18.4(口の分類)**

```text
PortClass := resource | signal | goal | selector
class : Port → PortClass
```

`[DEF]`

- `resource`: 維持に利用できる交換。
- `signal`: 環境との観測的結合。
- `goal`: 到達すべき外部状態・報酬・評価基準。
- `selector`: 設計者または個体群選択側の状態。

外部との交換一般を禁止せず、`goal/selector` から保護された個体内 sink への情報流を禁止する。

**定義 18.5(保護トレース非干渉)** §10.2 の `Trace` 自体を保護対象とし、`observe` の第一引数を外部評価状態 `g`、第二引数を個体入力 `i` と読んで、

```text
GoalNI(observe) :⇔ ∀g₁ g₂ i, observe(g₁,i) = observe(g₂,i)
```

とする。`[DEF]` これは §10.2 の `Pure(observe)` の引数を来歴型で精密化したもの。Trace には少なくとも `νΦ`、`𝐕`、求めの図式、作動選択、回復判定を含める。より小さい保護射影を使う場合は別の写像 `protect : Trace → ProtectedTrace` を明示し、上式の両辺へ合成する。

実行時に `g` を変える検査だけでは、学習・構築時に外部 goal を重みへ焼き込み、その後入力を切断した系を検出できない。そこで artifact の依存グラフに対し

```text
ProvenanceSafe(artifact) :⇔
  goal/selector 起点から
  {νΦ, 𝐕, seeking diagram, action, recovery criterion}
  への依存経路が存在しない
```

を別に要求する。`[DEF]` 依存を意図的に許す場合は「内生的」と表示せず、その来歴を証書へ残す。

**定義 18.6(内生的回復)**

```text
InternalFactor(recoveryObserve) :⇔
  ∃decide, ∀g i,
    recoveryObserve(g,i) = decide(internalView(i))

EndogenousRecovery :⇔
  Recoverable(B,A)
  ∧ InternalFactor(recoveryObserve)
  ∧ GoalNI(recoveryObserve)
  ∧ ProvenanceSafe(recoveryArtifact)
```

`[DEF]` `internalView` は閉包、構成、履歴等の対象内状態だけを読む。これは外部交換なしの回復を意味しない。`resource/signal` 結合を利用してよいが、回復の基準と方向は対象内状態を通じて因子化され、`goal/selector` の変更で変わらないことを意味する。

**定理 18.7** `GoalNI(observe)` なら任意の `g₁,g₂,i` について `M₄(observe(g₁,i)) ↔ M₄(observe(g₂,i))`。また §10.2 の `TraceSafe(μ)` なら変異前後で M4 を保存する。従って両条件のもとで、外部評価状態の変更と trace-safe な変異の双方に対して M4 は保存される。`[THM]`
> 証明: 前半は GoalNI の等式による書き換え、後半は定理 10.5。∎

静的な禁止語検査や引数一覧だけでは closure、alias、global state、control dependence を検出できない。完全保証を名乗る場合は二実行非干渉証明、情報流型、または同等の意味論的証書を要求する。`[META]`

### 18.4 証明証書と保証プロファイル

**定義 18.8(監査証書)**

```text
Certificate :=
  (claim, subjectFingerprint, abstractionVersion,
   dependencies, checker, trustBoundary, scope, result)

result := proved | refuted | bounded | observed | unknown
```

`[DEF]` `trustBoundary` は Lean core、Julia/Python shell、浮動小数点、データ取得を分ける。`Core=machine-checked` でも `AuditMap` がなければ現実実装についての保証はない。

**義務 18.9(監査橋)** 既存 certificate envelope を `Certificate` に写し、`AuditMap` の `fingerprint/scope/assumptions` が欠ける証書を `implementation-linked` と表示できないようにせよ。`PathLift` を持つ有限 `W₆` 証書と、may 抽象だけで存在ライブネスを誤認する反例を構成せよ。`[OBL]`

## §19. 構造生成・系譜・非周期性

### 19.1 保存射と生成事象の分離

`formal/ERIEC/OpenEvolution.lean` の二時間尺度開放系を抽象的に

```text
O := (Q_fast,Q_slow,Q_ext,step)
step : Config(O) → Set(Config(O))
```

と書く。厳密射は三成分写像 `f` と

```text
image(Config(f), step_O(c)) = step_O'(Config(f)(c))
```

を持つ。恒等射・合成・圏の三法則・到達保存は既存 Lean 核で証明済みである。`[THM]`

Transport(§5.5)とこの厳密射は、指定された構造または遷移意味論を保存する。新しい構造の生成を意味しない。

**定義 19.1(生成事象)** 保存射とは別に、開放系間の関係

```text
GenEvent : OpenSystem → OpenSystem → Prop
```

を置く。`GenEvent(O,O')` の証人は、親構成 `p:Config(O)`、子構成 `c:Config(O')`、生殖・変異記録、親子 heritage、必要な viability/M4 証書を束ねる依存構造体とする。`[DEF]` 生成事象は親状態から別の子開放系を生じる系譜関係であり、同一個体の通常遷移 `step` や一時的回復と同一視しない。

能力階層は

```text
OpenSystem ← ViableSystem ← ReplicativeSystem ← EvolutionarySystem
```

という忘却射の列として扱う。`[DEF]` 既存 `OpenEvolution.lean` が各構造と忘却を実装する。

### 19.2 意味同値を法とした新規性

単なる状態名・添字の増加を生成と数えないため、系の意味同値 `≈sem` を指定する。これは反射・対称・推移を満たし、用途に応じて厳密同型、双模倣、観測同値、または誤差予算付き lax 同値である。`[FLD]`

**定義 19.2(系譜と新規性)**

```text
Lineage :=
  (sys : ℕ → OpenSystem,
   event : ∀n, GenEvent(sys(n),sys(n+1)))

Fresh_sem(L) :⇔
  ∀n, ∃m≥n, ∀k<m, ¬(L.sys(m) ≈sem L.sys(k))

EventuallyPeriodic_sem(L) :⇔
  ∃N p, 0<p ∧ ∀n≥N, L.sys(n+p) ≈sem L.sys(n)
```

`[DEF]`

**定義 19.3(構造量の余終性)** 狭義半順序 `(Q,<)` と `≈sem` 不変な構造量 `q:OpenSystem→Q` に対し

```text
Cofinal_q(L) :⇔ ∀b:Q, ∃n, b < q(L.sys(n))
```

とする。`[DEF]` 一般半順序では通常の「上界を持たない」より強いので、`Unbounded` でなく `Cofinal` と呼ぶ。

**定理 19.4** `Cofinal_q(L) → Fresh_sem(L)`。また `Fresh_sem(L) → ¬EventuallyPeriodic_sem(L)`。`[THM]`
> 証明: 任意の有限接頭辞の `q` 値集合から極大元を一つ取り、余終性でそれより大きい値を持つ後続を得る。その値は接頭辞に存在しない。後半は最終周期を仮定すると、十分後の対象が有限個の同値類を反復することとの矛盾。∎

### 19.3 豊穣化と外部目的の分離

**定義 19.5(豊穣な子の到達可能性)**

開放系が指定 viability 構造を持つことを表す述語 `ViableObj : OpenSystem → Prop` を固定し、

```text
ProducesRicher(O,q) :⇔
  ViableObj(O) →
    ∃O', GenEvent(O,O') ∧ ViableObj(O') ∧ q(O) < q(O')
```

`[DEF]` これは変異空間に上昇辺が存在するという到達可能性仮定であり、実際の選択がその辺を取ることや、長期余終性を含意しない。後者には系譜公平性または確率核に関する追加仮定が要る。

`q` は read-only なメタ観測であり、個体内の `νΦ,𝐕,seeking diagram,action` に write-back しない。write-back する場合は §18 の `GoalNI` を失い、M4 違反の可能性を別途検査する。

### 19.4 実行の非周期性と生成の独立

二状態 `M_recur` で `b` の滞在長を `1,2,3,…` と増加させて `a` へ戻る実行は、タグ込みで非周期かつ帰還的だが、状態・系構造は二種類しかなく `Fresh_sem` でない。従って

```text
Recurrent ∧ Aperiodic ⇏ structural generation
```

である。逆に、新しい子を有限回生成した後に同一構造を反復する系譜は生成事象を持つが最終的に周期的である。非周期性と生成性は独立に証書化する。`[THM]`

### 19.4.1 増殖射(誕生半球)と DC→viable 翻訳 witness

本追補は `category/erie_c_axiom_categorical_memo.md §11-3` の増殖射判断を、Lean 実装順序に合わせて §19 の生成事象層へ接続する。2026-07-08 の批准により、次の三点を固定する。`[META]`

1. **生殖的分岐**を先に形式化し、発生的遷移(ontogeny)は後続の別命題とする。
2. `DC ⇒ viable` は対象層の一般定理ではなく、**一方向 translation witness** として実現する。逆向き `viable ⇒ DC` は引き続き禁止する。
3. 世代間の豊かさ遺伝は A-6 単段定理(`hinge_branch_pump`)とは別 statement とする。単段の豊かさ増幅と、増殖射沿いの遺伝契約を混同しない。

**定義 19.6a(DC 認証ユニットの開放系への忘却)** `Δ := (𝔽,s,dc : DC(𝔽,s))` を、局所時間を持つ開放系へ読む adapter を置く。

```text
dcToOpenSystem(Δ) : OpenSystem
dcViable(Δ)       : Config(dcToOpenSystem(Δ)) → Prop
```

## §20. 理論翻訳と用途別保証

### 20.1 翻訳の型

理論を署名、文、模型、充足関係の組として扱う。理論 `𝒯` から ERIE-C 構造言語 `𝓔` への翻訳は、単なる語彙表でなく文の翻訳と模型の還元が充足を保存する解釈

```text
Tr : 𝒯 → 𝓔
```

である。`[DEF]` institution morphism または同等の解釈として機械化する。

**定義 20.1(翻訳結果の分類)**

```text
structural   : ERIE-C の構造文へ充足保存的に翻訳済み
functional   : FM 等の操作的述語へ翻訳済み
bridge       : 構造から現象への追加仮定を使用
untranslated : 対応する文・模型写像が未定義
```

`[DEF]` `Conscious` という構造述語名を、現象的経験の証明と読まない。定理 7.3 と §9.2 の橋が境界を与える。

### 20.2 保証プロファイル

単一の `ERIEC-certified : Bool` を置かず、証書の直積として

```text
Core        : none | model | machineChecked
Audit       : none | bounded | simulationSound | implementationLinked
Viability   : none | recoverable | possibleLive | fairLive | boundedLive
Generative  : none | observed | fresh | cofinal
Translation : none | glossary | satisfactionPreserving | conservative
Phenomenal  : notCertified
```

を用いる。`[DEF]`

- AI監査には `Audit≥simulationSound`、来歴型、`GoalNI`、到達成分上の `NoT*` が必要。
- 人工生命・エージェント設計には `Viability≥fairLive` と回復包絡・scheduler 証書を推奨。
- 非周期的構造生成を名乗るには `Aperiodic` と `Generative≥fresh` を別々に要求。
- 意識理論比較には `Translation≥satisfactionPreserving` と bridge 依存追跡が必要。
- 美学的使用は非規範的 readout であり、個人の価値・経験・健康を測定したとは表示しない。

### 20.3 決定可能性の境界

有限表示された模型では、到達、`NoT*_Init`、有限閉包、`W₆` の lasso 標準形、有限証書の依存関係を有限手続きで検査できる。一方、任意プログラムの全実行ライブネス、無限非周期性、余終的生成、実装と模型の完全意味論同値、現象的経験は一括して有限時間判定可能とは主張しない。`[META]`

一般系では、帰納的不変量、余帰納的 simulation、ランキング／反ランキング関数、path-lifting、有限地平観測のいずれを用いたかを証書に記録し、`proved/bounded/observed/unknown` を区別する。

**義務 20.2(翻訳・保証台帳)** FM1–FM4 について少なくとも一つの外部理論から充足保存的翻訳を構成するか、構成不能な項目を `untranslated` と証明依存グラフに記録せよ。certificate envelope が保証軸を個別に保持し、ある軸の強さで別軸を自動昇格できないことを型で保証せよ。`[OBL]`
