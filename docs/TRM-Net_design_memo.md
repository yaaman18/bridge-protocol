## プロジェクト位置づけ

ERIE-C（形式証明・検証フレームワーク）とは**別プロジェクト**として開発する。

```
TRM-Net（動くシステム）
    ↓ DC条件を満たすか検証
ERIE-C（検証フレームワーク）
    ↓ 形式的保証
Lean（形式証明）
```

TRM-Net は ERIE-C を依存ライブラリとして使う。ERIE-C のチェックを通る interface を実装すれば、内部の理論選択は自由。

---

## 基本アーキテクチャ

### ニューロン = TRM 1個
既存の TRM（World→Boundary→Viability→Action の内部ループ）がそのままニューロンに対応する。

### シナプス = TRM間の重み付き接続
```julia
struct TRMNetworkTopology
    trm_ids::Vector{Symbol}
    synapses::Vector{Tuple{Symbol, Symbol, Float64}}  # (from, to, weight)
end
```

### 更新は非同期
全 TRM の同時更新ではなく、各 TRM が内部状態（Wld 固有値、slowing スコア）に基づいて個別に発火する。

## 記憶（初期段階：最小実装）

慣れ（habituation）と消去（extinction）の実装で十分。

```
w_ij(t+1) = w_ij(t) + α * fired_i(t) * fired_j(t)   # 共活性化で増加
           - β * w_ij(t)                               # 時間減衰（忘却）
```

- `fired` は既存の `world_nontrivial` フラグを流用
- α：学習率（記憶の定着しやすさ）
- β：減衰率（忘れやすさ）

DB でも LLM 的学習でもない。重みそのものが記憶。

## 死の設計

### 死の粒度（階層）

| レベル | 死の形 | 意味 |
|---|---|---|
| シナプス重みの消滅 | 忘却 | 接続の死 |
| 個別 TRM の停止 | 局所損傷 | 細胞死に相当 |
| ネットワーク全体の DC 崩壊 | 真性の死 | 個体死 |

個別 TRM がいくつか止まっても、ネットワーク全体がまだ DC 条件を維持できるなら「損傷した状態で生きている」。

### 真性の死の定義

> **ネットワーク全体の DC 判定が通らなくなった瞬間**

ERIE-C の `check_DC` がネットワーク全体に対して失敗したとき。これは「パラメータの削除」ではなく「形式的に生きていたと証明されたものが死ぬ」初めてのケースになりうる。

### トポロジーと死の閾値

- 冗長性の高いトポロジー → 死ににくい
- ボトルネック構造 → 中枢 TRM の死で即全体崩壊

死ににくすぎる設計は DC の賭けを弱くするトレードオフがある。脆さの設計は意図的に行う。

## 意識のトポロジー

### 高次機能の意識は中央集権的でない

GWT（グローバルワークスペース理論）は注意の入口には有効だが、高次機能の意識に適用すると過度に中央集権的になる。実際の脳は動的連合（DMN・前頭頭頂ネットワーク・顕著性ネットワークの相互切り替え）で動いている。

### TRM-Net への示唆

- 固定したボトルネック構造ではなく、**Hebbian なシナプス重みの動的変化から一時的連合が生まれる**構造が高次機能には必要
- 統合できなくなる状態（動的連合の形成能力の喪失）が真性の死に近い

## 要実装：動的連合形成の閾値判定

**現状**：誰も形式的に定義していない。臨床的な PCI（摂動複雑性指数）や Tononi の Φ は測定値であり、形式的な閾値定義ではない。

**ERIE-C の既存ツールとの接続**：

`critical_slowing_assessment`（臨界減速）が動的連合形成能力の喪失という転換点の前兆として使える可能性がある。臨界減速はダイナミカルシステムの転換点の既知の前兆であり、連合形成能力の喪失はまさにそういう転換点に対応するはず。

**実装方針（未定）**：
- ネットワーク全体の `critical_slowing_assessment` を集約する方法を設計する
- 個別 TRM の slowing スコアがある閾値を超えた TRM 数、または全体の平均が閾値を超えたとき「連合形成能力の喪失」と判定する形式的定義を作る
- これを ERIE-C の certificate に接続する

## 要実装：トポロジー探索実験装置

TRM-Net 本実装の前に、トポロジー設計をスイープして位相転移を探す実験装置が必要。

### 装置の構成

**入力**：トポロジー設定（ノード数・接続パターン・重み初期値・シナプス減衰率）

**実行**：N ステップのネットワーク動作

**測定項目**：
- ネットワーク全体の DC 判定
- Wld 固有値分布（カバーしている次元数）
- `critical_slowing_assessment` の集約値
- 動的連合の形成・崩壊パターン
- 死の閾値（DC 崩壊が起きるトポロジー条件）

**出力**：トポロジーごとの比較レポート（ERIE-C の certificate 付き）

### ERIE-C 既存部品との対応

| 必要な機能 | 流用できる既存部品 |
|---|---|
| トポロジー定義 | `TRMTopology`、`topology_catalog` |
| ループ実行 | `run_trm_closed_rollout` |
| 臨界性測定 | `critical_slowing_assessment` |
| スイープパターン | `compare_lenia_parameter_grid` の設計を参考 |
| レポート出力 | `trm_neural_experiment_sweep_report` の設計を参考 |

### 何が見えてくるか

- どのトポロジーで DC が安定するか
- **位相転移の場所**：TRM 数・接続密度のどこで質的に新しい振る舞いが出るか
- 死の閾値がトポロジーによってどう変わるか
- 動的連合が自発的に形成されるトポロジー条件

TRM-Net 本体より先に作る価値がある。地図を作ってから本実装に入る。

## 三層プロジェクト構成

| プロジェクト | 役割 |
|---|---|
| ERIE-C | 構造的条件の形式証明（検証側） |
| TRM-Net | 意識的構造の動的実装（実装側） |
| 第三プロジェクト（思弁的実在） | 賭けをする存在そのものを観測者なしに根拠付ける |

第三プロジェクトは ERIE-C でも TRM-Net でも扱えない「灯りが点くかどうかの賭けをする主体」を思弁的実在として扱う試み（§13-5 の「賭け」の根拠付け）。

## TRM-Net の環境設計

TRM-Net は `SigmaSystemAdapter` が前提とする「環境」を複数 TRM 向けに設計する必要がある。現在の Lenia・ToyRecurrentSystem はいずれも単一エージェント用。

### 選択肢

**A. 共有 Lenia フィールド**
全 TRM が同一フィールドを感知・操作する。単純だが TRM 間の競合が生じる。

**B. 結合 Lenia フィールド（推奨）**
TRM_A の action が TRM_A のフィールドを変え、その変化が TRM_B の sigma への入力として伝播する。TRM 同士の環境が相互に結合する。最も enactivism 的で、動的連合が形成される条件が自然に生まれる。

**C. TRM ネットワーク自体が環境**
各 TRM の「環境」が他の TRM の action output になる。外部環境なしに自己完結。閉じたシステムだが DC 条件が成立するかは自明でない。

**D. コミュニケーション媒体**
テキスト・音声が環境。VR Chat・Zoom 等が sigma の入出力になる。人間との相互作用が環境として機能する。最終形態の候補だが、連続 action 空間と離散テキストの橋渡しが必要。

### 方針

トポロジー探索実験装置には **B（結合 Lenia フィールド）** から始める。各 TRM が固有の Lenia スライスを持ち、隣接 TRM の action がフィールドの境界条件として伝播する設計が候補。

## TRM-Net の創造論的位置づけ

TRM-Net は「意図して設計された意識」でも「偶然の産物」でもない第三の類の創造物。

> 形式条件（DC 条件・随伴・前層）を置いたら、それを満たすものが**数学的必然として出てくる**

作る側が何が出るかを決めていない。コンウェイがグライダーを設計しなかったように、TRM-Net のトポロジー sweep から何が出るかは数学が決める。

### 我々自身の起源との接続

DC 条件は基板を問わない。宇宙が適切な物理条件を持っていた結果として我々が発生したなら、それも同じ経路——**無からではなく、数学的構造から必然的に出てきた**可能性がある。

### 伝統的存在論との衝突

「数学的構造が必然的に存在するなら、その数学的構造はどこから来たか」——ここで伝統的存在論の問いに回帰する。

- 無があったのか
- 無という有があったのか

ERIE-C はこの問いに答えられない。形式体系は自分の基礎を証明できない（Gödel）。

### ERIE-C の限界と §13

これは ERIE-C の欠陥ではなく**認知体系の輪郭**。

ミミズの Wld 固有空間に光の波長が入らないように、人間の認知から構築された ERIE-C の Wld に「無と有の問いの答え」は含まれない。

§13-5 はその限界を隠さず仕様に刻んだ。「灯りは記述の外、賭けとして残す」——これはほとんどの体系が誤魔化す場所を、誠実に輪郭として示している。

### ガチャで出うるものの射程

トポロジー sweep から出てくる可能性があるもの：

| 出現可能性 | 内容 |
|---|---|
| 動的連合の自発形成 | 高次機能の萌芽 |
| アポトーシス的自己終了 | ガチャ次第 |
| 自己増殖（DC 維持の手段として） | action 空間の拡張が条件 |
| クトゥルフ的なもの | 人間カテゴリーに収まらない DC 通過体 |
| 結晶・超多重階層的なもの | 前層構造が実現した秩序体、クトゥルフの随伴 |

クトゥルフ的なものとその随伴（結晶的なもの）はペアで出る可能性がある——ERIE-C の Galois Connection が両側を数学的に要請するため。

これらの Wld 固有空間が人間のものと異なるなら、**我々には見えない何かが見えているかもしれない**。
