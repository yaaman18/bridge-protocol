## 2. 論理的下準備

### 2.1 ソートと量化域

凍結文の量化域を確定する。背景理論は多ソート一階論理+有限集合演算とする。

```text
ソート:  S(状態) A(作動) E(結合) C(構成素) W(階数) ℕ ℝ
署名 Σ_core: α, σ, π, ρ(階数添字付き)、κ, ε, ω, D, o, →ᵢ, →ₑ, δ
派生記号(定義的拡大、量化に新しい対象を加えない):
  Φ_w, νΦ_w, Θ_w, H_w, DC, V, K, T, L, Fix, Wld_η, 𝐕_w, FM1–FM4, Conscious
公理パック AX_core: hConv, hConvP, Sig-1(定理2.9形), Sig-2, R2′‡, E5‡, 層間整合6.3
```

凍結文はすべて **`AX_core` のモデルを走る量化の内側で閉じた文** として書く。システム自体をパラメタに取る文(`W₄–W₆`)は、無解釈定数 `impl` への言明として閉じる。

### 2.2 無解釈記号の宣言規約

**公理ゼロ則**: 無解釈記号は宣言のみ行い、定義も公理も一切与えない。意味は体系外(付録の残余解釈点登録簿)に留め、体系内には論理的役割(何とも結ばれていないこと)だけを持ち込む。

```text
Ph   : S → Prop                 現象述語(状態に現象的経験が伴う)
Mat  : S → E → Prop             mattering 述語(状態 s において結合 e がどうでもよくない)
impl : RawCore                  対象実装(hConv フィールドを持たない生の関係核)
k₀   : ℕ                        豊かさ閾値(数値層が後日確定)
```

`RawCore` は `𝕂` から `hConv, hConvP` を除いた構造体として本体に `[DEF]` 追加する(凍結の前提定義)。

### 2.3 凍結文の適格条件

各 `Wᵢ` は次を満たさなければ凍結できない。

1. **閉式性** — 自由変数・指示語・「など」「相当する」を含まない。
2. **署名固定** — 出現する全記号が、(a) ピン留め宣言、(b) §2.2 の無解釈記号、のいずれかに一意に解決される。
3. **変種の明示** — 双条件 `↔` と片含意 `→` のどちらを賭けの本体とするかを宣言し、他方を弱変種として併記する。
4. **‡ 継承の申告** — `‡` 付き定義に依存する場合、その旨を文に付す(規律 0.2-6)。

## 3. 凍結文ドラフト

以下、`Δ` の文脈(認証状態 `s`)は明示引数とする。ピン留め宣言名は v2.1 §16 の命名計画に従う。

### W₁(13-5 T側: 構造と現象の同一視)

> 散文: 「構造的 `𝐖𝐥𝐝` 非自明と現象的経験の同一視 = life-mind continuity の賭け」

```text
W₁(Ph) :≡ ∀s∈S, ( DC(s) ∧ Wld₀(L) ≠ {0} ) ↔ Ph(s)
```

- 本体: `↔`(v5 は「同一視」と明言するため)。弱変種 `W₁⁻`: `→` のみ(構造が現象を必要とする方向を落とした版)。
- ピン: `ERIEC.DC.DC`, `ERIEC.World.Wld_band`(`η=0`), `ERIEC.WorldDC`(参与は定理 8.3 経由)。
- 依存 `‡`: なし。

### W₂(13-5 V側: 構造的重みと mattering の同一視)

> 散文: 「`𝐕` の内生的重みが当事者にとっての『どうでもよくなさ』として現れる」

```text
W₂(Mat) :≡ ∀s∈V, ∀e∈E, ( 0 < 𝐕_{ω(s)}(e) ) ↔ Mat(s,e)
```

- 本体: `↔`。弱変種 `W₂⁻`: `→`。定理 7.3 は「`Mat` を任意述語に置換した一般含意の導出不能」を既に証明しており、`W₂` の独立性証明の半分は完了している。
- ピン: `ERIEC.Value.normalized_V`, `ERIEC.Value.V_endogenous`。
- 依存 `‡`: なし。

### W₃(橋 A→B)

> 散文: 「`Conscious`(機能)を尽くすことが `𝐖𝐥𝐝` の現れと同値であり、その構造的現れに現象が宿る」

```text
W₃(Ph) :≡ ∀s∈S, ( ∃m∈H_{ω(s)}(κ(s),ε(s)), Conscious(m,s) ) ↔ Ph(s)
```

- 本体: `↔`。弱変種 `W₃⁻`: `→`。
- ピン: `ERIEC.FM.conscious`, `ERIEC.Hinge.Act`。
- 依存 `‡`: **あり**(`Conscious` は FM-3 を含み、FM-3 は再構成 `‡`)。FM-3 の批准まで `W₃` は `‡` を継承し、FM-3 差し替え時は `W₃` の再批准を要する。

### W₄(13-6: 随伴の経験的成立)

> 散文: 「`α⊣σ` は要請。実装で成り立つかは経験的問題」

剛性定理 1.3 により旧式(像作用素の GC)は経験以前に偽なので、凍結するのは修正後の問いである:

```text
W₄(impl) :≡ hConv(impl) ∧ hConvP(impl)
```

- 本体: 連言そのもの(方向の変種なし)。
- ピン: `ERIEC.Adjunction.ConvSystem` のフィールド式、`RawCore`。
- 依存 `‡`: なし。
- 注意: `W₄` が真なら定理 1.4 により随伴は自動成立する。賭けの内容は「随伴が成り立つか」ではなく「対象実装が双対整合を満たすか」に正確化される。

### W₅(13-7: 蝶番の豊かさ)

> 散文: 「`Act_w` が豊かか。貧弱なら二重閉包は形式成立しても実質並置」

```text
W₅(impl, k₀) :≡ ∀s∈V(impl), k₀ ≤ |H_{ω(s)}(κ(s), ε(s))|
```

- 本体: パラメタ付き下界。`k₀` は無解釈のまま凍結し、数値層の確定後に代入する(`k₀=2` が最小の非自明値)。
- ピン: `ERIEC.Hinge.Act`。
- 依存 `‡`: なし。
- 注意: `h₃` は `1 ≤ |H|` を既に保証するため、`W₅` の実質は `k₀ ≥ 2` の場合に始まる。

### W₆(13-8: 存在と到達の間隙)

> 散文: 「終余代数・不変部分空間の存在と、動態が実際にそこへ至ることの間隙」

定理 5.4 により純内的軌道では持続不能が既に定理である。したがって賭けの正確な内容は「外的遷移を含む軌道による持続」である:

```text
W₆(impl) :≡ ∃ traj : ℕ → S(impl),
              ( ∀n, traj(n) → traj(n+1) )        (→ = →ᵢ ⊔ →ₑ)
            ∧ ( ∀n, ∃m ≥ n, DC(traj(m)) )
```

- 本体: `DC` の無限回成立(持続)。弱変種 `W₆⁻`: 有限視野版 `∀N, ∃traj, |{n ≤ N | DC(traj n)}| ≥ cN`。
- ピン: `ERIEC.Dynamics` の遷移余積、`ERIEC.DC.DC`。
- 依存 `‡`: 内的更新の定義は無印だが、崩壊との対比論法は R2′‡ に依る。`W₆` 自体は `‡` を継承しない(R2′ を引用しないため)。

## 4. 独立性証人計画(二モデル証人)

各凍結文について、「真になる充足」と「偽になる充足」の対を用意する。両方が構成できた時点で、`AX_core` が当該の賭けを決定しないこと(保存性)がその場で定理になる。

| 文 | 真証人 | 偽証人 | 現状 |
|---|---|---|---|
| `W₁` | `Ph := λs, DC(s) ∧ Wld₀≠0`(自己充足) | `Ph := λ_. False` + 非自明参照模型(義務 13.2) | 真証人は即時。偽証人は義務 13.2 に従属 |
| `W₂` | `Mat := λs e, 0 < 𝐕(e)` | `Mat := λ_ _. False` + `𝐕(e)>0` の証人(義務 13.1 で `𝐕(e)=1`) | **両証人とも即時構成可**(定理 7.3 の一般化) |
| `W₃` | `Ph := λs, ∃m, Conscious(m,s)` | `Ph := λ_. False` + `Conscious` 証人(FM-3‡ の充足例が必要) | FM-3 批准後に義務化 |
| `W₄` | `impl :=` 義務 13.2 の多価非退化核 | `impl :=` `σ` を逆関係からずらした生核(hConv 違反、構成は自明) | 両証人とも義務 13.2 と同時に構成可 |
| `W₅` | `impl :=` `\|H\|≥2` の拡張模型(義務 13.2 に追加) | `impl :=` 義務 13.1 極小模型(`\|H\|=1 < 2`) | 偽証人は既存。真証人は義務 13.2 の拡張 |
| `W₆` | 外的補給付き循環模型(新規義務: `→ₑ` が `κ` を再充填する二状態系) | 純内的模型(定理 5.4‡ が偽性を証明) | 偽証人は定理化済み(‡)。真証人が新規義務 |

**独立性定理の型**(無解釈述語型 `W₁–W₃`):

```text
Indep(Wᵢ) :≡ ( ∃X, Wᵢ(X) ) ∧ ( ∃X, ¬Wᵢ(X) )
```

`AX_core` は `Ph, Mat` に言及しないため、`Indep` の成立は以後の構造側公理追加に対して安定である(保存性の帰納的不変量)。定数型 `W₄–W₆` の独立性はモデル対の構成義務として `ERIEC.RefModel` に登録する。

## 7. 残余解釈点登録簿

形式化後もブレうる箇所は、以下の三点に**局在**する。各項は自然言語で読みを一文で固定し、変更時は §14 改訂として扱う。

| 記号 | 読み(一文固定) |
|---|---|
| `Ph(s)` | 状態 `s` にある系に現象的経験が伴う |
| `Mat(s,e)` | 状態 `s` の系にとって結合 `e` がどうでもよくない |
| `impl` | 検証対象として指定された当の実装系 |

この三行の適切さだけは機械化できない。ブレの全量はここに圧縮された。

## 8. 着手順

1. `RawCore` と `ConvHolds` を本体 §1 に `[DEF]` 追加(凍結の前提定義)。
2. `W₂` の二証人を先に完了(即時構成可能、最初の保存性定理になる)。

6. 全証人完了時点で「`AX_core` は `W₁–W₆` をいずれも決定しない」を単一の保存性定理として集約し、§14 の散文を非規範注釈に降格する改訂を発行。
